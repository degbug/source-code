  31 /**
  32  * Defines generic behaviour for the server
  33  * part of a connector or an adaptor. Most connectors or adaptors extend CommunicatorServer
  34  * and inherit this behaviour. Connectors or adaptors that do not fit into this model do not extend
  35  * CommunicatorServer.
  36  * 
  37  * An CommunicatorServer is an active object, it listens for client requests
  38  * and processes them in its own thread. When necessary, a CommunicatorServer
  39  * creates other threads to process multiple requests concurrently.
  40  * 
  41  * A CommunicatorServer object can be stopped by calling the stop
  42  * method. When it is stopped, the CommunicatorServer no longer listens to client
  43  * requests and no longer holds any thread or communication resources.
  44  * It can be started again by calling the start method.
  45  * 
  46  * A CommunicatorServer has a state property which reflects its
  47  * activity.
  48  * 
  49  * 

  50  * CommunicatorServer            State
  51  * stopped          OFFLINE
  52  * starting         STARTING
  53  * running          ONLINE
  54  * stopping         STOPPING
  55  * 
  56  * 
  57  * The STARTING state marks the transition from OFFLINE to
  58  * ONLINE.
  59  * 
  60  * The STOPPING state marks the transition from ONLINE to
  61  * OFFLINE. This occurs when the CommunicatorServer is
  62  * finishing or interrupting active requests.
  63  * 
  64  * A CommunicatorServer may serve several clients concurrently. The
  65  * number of concurrent clients can be limited using the property
  66  * maxActiveClientCount. The default value of this property is
  67  * defined by the subclasses.
  68  * 
  69  * When a CommunicatorServer is unregistered from the MBeanServer,
  70  * it is stopped automatically.
  71  *
  72  * This API is a Sun Microsystems internal API  and is subject
  73  * to change without notice.
  74  */
  75
