  38  * General notes to translators:
  39  *
  40  * This file contains error and warning messages related to XPath Error
  41  * Handling.
  42  *
  43  *  1) Xalan (or more properly, Xalan-interpretive) and XSLTC are names of
  44  *     components.
  45  *     XSLT is an acronym for "XML Stylesheet Language: Transformations".
  46  *     XSLTC is an acronym for XSLT Compiler.
  47  *
  48  *  2) A stylesheet is a description of how to transform an input XML document
  49  *     into a resultant XML document (or HTML document or text).  The
  50  *     stylesheet itself is described in the form of an XML document.
  51  *
  52  *  3) A template is a component of a stylesheet that is used to match a
  53  *     particular portion of an input document and specifies the form of the
  54  *     corresponding portion of the output document.
  55  *
  56  *  4) An element is a mark-up tag in an XML document; an attribute is a
  57  *     modifier on the tag.  For example, in <elem attr='val' attr2='val2'>
  58  *     "elem" is an element name, "attr" and "attr2" are attribute names with
  59  *     the values "val" and "val2", respectively.
  60  *
  61  *  5) A namespace declaration is a special attribute that is used to associate
  62  *     a prefix with a URI (the namespace).  The meanings of element names and
  63  *     attribute names that use that prefix are defined with respect to that
  64  *     namespace.
  65  *
  66  *  6) "Translet" is an invented term that describes the class file that
  67  *     results from compiling an XML stylesheet into a Java class.
  68  *
  69  *  7) XPath is a specification that describes a notation for identifying
  70  *     nodes in a tree-structured representation of an XML document.  An
  71  *     instance of that notation is referred to as an XPath expression.
  72  *
  73  *  8) The context node is the node in the document with respect to which an
  74  *     XPath expression is being evaluated.
  75  *
  76  *  9) An iterator is an object that traverses nodes in the tree, one at a time.
  77  *
  78  *  10) NCName is an XML term used to describe a name that does not contain a
  79  *     colon (a "no-colon name").
  80  *
  81  *  11) QName is an XML term meaning "qualified name".
