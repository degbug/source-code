  44 /**
  45  * This interface permits a DOM implementer to supply one or more
  46  * implementations, based upon requested features and versions, as specified
  47  * in DOM
  48  * Features. Each implemented DOMImplementationSource object is
  49  * listed in the binding-specific list of available sources so that its
  50  * DOMImplementation objects are made available.
  51  * See also the Document Object Model (DOM) Level 3 Core Specification.
  52  * @since DOM Level 3
  53  */
  54 public interface DOMImplementationSource {
  55     /**
  56      *  A method to request the first DOM implementation that supports the
  57      * specified features.
  58      * @param features  A string that specifies which features and versions
  59      *   are required. This is a space separated list in which each feature
  60      *   is specified by its name optionally followed by a space and a
  61      *   version number.  This method returns the first item of the list
  62      *   returned by getDOMImplementationList.  As an example,
  63      *   the string "XML 3.0 Traversal +Events 2.0" will
  64      *   request a DOM implementation that supports the module "XML" for its
  65      *   3.0 version, a module that support of the "Traversal" module for
  66      *   any version, and the module "Events" for its 2.0 version. The
  67      *   module "Events" must be accessible using the method
  68      *   Node.getFeature() and
  69      *   DOMImplementation.getFeature().
  70      * @return The first DOM implementation that support the desired
  71      *   features, or null if this source has none.
  72      */
  73     public DOMImplementation getDOMImplementation(String features);
  74 
  75     /**
  76      * A method to request a list of DOM implementations that support the
  77      * specified features and versions, as specified in .
  78      * @param features A string that specifies which features and versions
  79      *   are required. This is a space separated list in which each feature
  80      *   is specified by its name optionally followed by a space and a
  81      *   version number. This is something like: "XML 3.0 Traversal +Events
  82      *   2.0"
  83      * @return A list of DOM implementations that support the desired
  84      *   features.
  85      */
  86     public DOMImplementationList getDOMImplementationList(String features);
  87 
  88 }
