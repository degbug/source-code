  32 /**
  33  * This class represents a default implementation for
  34  * javax.security.auth.Policy.
  35  *
  36  *  This object stores the policy for entire Java runtime,
  37  * and is the amalgamation of multiple static policy
  38  * configurations that resides in files.
  39  * The algorithm for locating the policy file(s) and reading their
  40  * information into this Policy object is:
  41  *
  42  * 
    
  43  * 
  1. 
  44  *   Loop through the security properties,
  45  *   auth.policy.url.1, auth.policy.url.2, ...,
  46  *   auth.policy.url.X".
  47  *   Each property value specifies a URL pointing to a
  48  *   policy file to be loaded.  Read in and load each policy.
  49  *
  50  * 
  2. 
  51  *   The java.lang.System property java.security.auth.policy
  52  *   may also be set to a URL pointing to another policy file
  53  *   (which is the case when a user uses the -D switch at runtime).
  54  *   If this property is defined, and its use is allowed by the
  55  *   security property file (the Security property,
  56  *   policy.allowSystemProperty is set to true),
  57  *   also load that policy.
  58  *
  59  * 
  3. 
  60  *   If the java.security.auth.policy property is defined using
  61  *   "==" (rather than "="), then ignore all other specified
  62  *   policies and only load this policy.
  63  * 
  64  *
  65  * Each policy file consists of one or more grant entries, each of
  66  * which consists of a number of permission entries.
  67  *
  68  * 
    
  69  *   grant signedBy "alias", codeBase "URL",
  70  *         principal principalClass "principalName",
  71  *         principal principalClass "principalName",
  72  *         ... {
  73  *
  74  *     permission Type "name "action",
  75  *         signedBy "alias";
  76  *     permission Type "name "action",
  77  *         signedBy "alias";
  78  *     ....
  79  *   };
  80  * 
  81  *
  82  * All non-bold items above must appear as is (although case
  83  * doesn't matter and some are optional, as noted below).
  84  * Italicized items represent variable values.
  85  *
  86  *  A grant entry must begin with the word grant.
  87  * The signedBy and codeBase
  88  * name/value pairs are optional.
  89  * If they are not present, then any signer (including unsigned code)
  90  * will match, and any codeBase will match.  Note that the
  91  * principal name/value pair is not optional.
  92  * This Policy implementation only permits
  93  * Principal-based grant entries.  Note that the principalClass
  94  * may be set to the wildcard value, *, which allows it to match
  95  * any Principal class.  In addition, the principalName
  96  * may also be set to the wildcard value, *, allowing it to match
  97  * any Principal name.  When setting the principalName
  98  * to the *, do not surround the * with quotes.
  99  *
 100  *  A permission entry must begin with the word permission.
 101  * The word Type in the template above is
 102  * a specific permission type, such as java.io.FilePermission
 103  * or java.lang.RuntimePermission.
 104  *
 105  *  The "action" is required for
 106  * many permission types, such as java.io.FilePermission
 107  * (where it specifies what type of file access that is permitted).
 108  * It is not required for categories such as
 109  * java.lang.RuntimePermission
 110  * where it is not necessary - you either have the
 111  * permission specified by the "name"
 112  * value following the type name or you don't.
 113  *
 114  *  The signedBy name/value pair for a permission entry
 115  * is optional. If present, it indicates a signed permission. That is,
 116  * the permission class itself must be signed by the given alias in
 117  * order for it to be granted. For example,
 118  * suppose you have the following grant entry:
 119  *
 120  * 
    
 121  *   grant principal foo.com.Principal "Duke" {
 122  *     permission Foo "foobar", signedBy "FooSoft";
 123  *   }
 124  * 
 125  *
 126  *  Then this permission of type Foo is granted if the
 127  * Foo.class permission has been signed by the
 128  * "FooSoft" alias, or if Foo.class is a
 129  * system class (i.e., is found on the CLASSPATH).
 130  *
 131  *  Items that appear in an entry must appear in the specified order
 132  * (permission, Type, "name", and
 133  * "action"). An entry is terminated with a semicolon.
 134  *
 135  *  Case is unimportant for the identifiers (permission,
 136  * signedBy, codeBase, etc.) but is
 137  * significant for the Type
 138  * or for any string that is passed in as a value. 
 139  *
 140  *  An example of two entries in a policy configuration file is
 141  * 
    
 142  *   // if the code is comes from "foo.com" and is running as "Duke",
 143  *   // grant it read/write to all files in /tmp.
 144  *
 145  *   grant codeBase "foo.com", principal foo.com.Principal "Duke" {
 146  *              permission java.io.FilePermission "/tmp/*", "read,write";
 147  *   };
 148  *
 149  *   // grant any code running as "Duke" permission to read
 150  *   // the "java.vendor" Property.
 151  *
 152  *   grant principal foo.com.Principal "Duke" {
 153  *         permission java.util.PropertyPermission "java.vendor";
 154  * 
 155  *
 156  *  This Policy implementation supports
 157  * special handling for PrivateCredentialPermissions.
 158  * If a grant entry is configured with a
 159  * PrivateCredentialPermission,
 160  * and the "Principal Class/Principal Name" for that
 161  * PrivateCredentialPermission is "self",
 162  * then the entry grants the specified Subject permission to
 163  * access its own private Credential.  For example,
 164  * the following grants the Subject "Duke"
 165  * access to its own a.b.Credential.
 166  *
 167  * 
    
 168  *   grant principal foo.com.Principal "Duke" {
 169  *      permission javax.security.auth.PrivateCredentialPermission
 170  *              "a.b.Credential self",
 171  *              "read";
 172  *    };
 173  * 
 174  *
 175  * The following grants the Subject "Duke"
 176  * access to all of its own private Credentials:
 177  *
 178  * 
    
 179  *   grant principal foo.com.Principal "Duke" {
 180  *      permission javax.security.auth.PrivateCredentialPermission
 181  *              "* self",
 182  *              "read";
 183  *    };
 184  * 
 185  *
 186  * The following grants all Subjects authenticated as a
 187  * SolarisPrincipal (regardless of their respective names)
 188  * permission to access their own private Credentials:
 189  *
 190  * 
    
 191  *   grant principal com.sun.security.auth.SolarisPrincipal * {
 192  *      permission javax.security.auth.PrivateCredentialPermission
 193  *              "* self",
 194  *              "read";
 195  *    };
 196  * 
 197  *
 198  * The following grants all Subjects permission to access their own
 199  * private Credentials:
 200  *
 201  * 
    
 202  *   grant principal * * {
 203  *      permission javax.security.auth.PrivateCredentialPermission
 204  *              "* self",
 205  *              "read";
 206  *    };
 207  *

 212  *
 213  * @see java.security.CodeSource
 214  * @see java.security.Permissions
 215  * @see java.security.ProtectionDomain
 216  * @see java.security.Security security properties
 217  */

 222     private final sun.security.provider.AuthPolicyFile apf;

 224     /**
 225      * Initializes the Policy object and reads the default policy
 226      * configuration file(s) into the Policy object.
 227      */
 228     public PolicyFile() {

 232     /**
 233      * Refreshes the policy object by re-reading all the policy files.

 235      *

 237      * @exception SecurityException if the caller doesn't have permission
 238      *          to refresh the Policy.

 240     @Override
 241     public void refresh() {
 242         apf.refresh();
 243     }

 245     /**
 246      * Examines this Policy and returns the Permissions granted
 247      * to the specified Subject and CodeSource.
 248      *
 249      *  Permissions for a particular grant entry are returned
 250      * if the CodeSource constructed using the codebase and
 251      * signedby values specified in the entry implies
 252      * the CodeSource provided to this method, and if the
 253      * Subject provided to this method contains all of the
 254      * Principals specified in the entry.
 255      *
 256      *  The Subject provided to this method contains all
 257      * of the Principals specified in the entry if, for each
 258      * Principal, "P1", specified in the grant entry
 259      * one of the following two conditions is met:
 260      *
 261      * 
 262      * 
      
 263      * 
    1.  the Subject has a
      2. 
 264      *      Principal, "P2", where
 265      *      P2.getClass().getName() equals the
 266      *      P1's class name, and where
 267      *      P2.getName() equals the P1's name.
 268      *
 269      * 
    2.  P1 implements
      3. 
 270      *      com.sun.security.auth.PrincipalComparator,
 271      *      and P1.implies the provided Subject.
 272      * 
 273      *
 274      *  Note that this Policy implementation has
 275      * special handling for PrivateCredentialPermissions.
 276      * When this method encounters a PrivateCredentialPermission
 277      * which specifies "self" as the Principal class and name,
 278      * it does not add that Permission to the returned
 279      * PermissionCollection.  Instead, it builds
 280      * a new PrivateCredentialPermission
 281      * for each Principal associated with the provided
 282      * Subject.  Each new PrivateCredentialPermission
 283      * contains the same Credential class as specified in the
 284      * originally granted permission, as well as the Class and name
 285      * for the respective Principal.
 286      *
 287      * 
 288      *
 289      * @param subject the Permissions granted to this Subject
 290      *          and the additionally provided CodeSource
 291      *          are returned. 
 292      *
 293      * @param codesource the Permissions granted to this CodeSource
 294      *          and the additionally provided Subject
 295      *          are returned.
 296      *
 297      * @return the Permissions granted to the provided Subject
 298      *          CodeSource.
 299      */
 300     @Override
 301     public PermissionCollection getPermissions(final Subject subject,
 302                                                final CodeSource codesource) {
 303         return apf.getPermissions(subject, codesource);
 304     }
 305 }
