  32 /**
  33  * A representation of the SOAP header
  34  * element. A SOAP header element consists of XML data that affects
  35  * the way the application-specific content is processed by the message
  36  * provider. For example, transaction semantics, authentication information,
  37  * and so on, can be specified as the content of a SOAPHeader
  38  * object.
  39  * 

  40  * A SOAPEnvelope object contains an empty
  41  * SOAPHeader object by default. If the SOAPHeader
  42  * object, which is optional, is not needed, it can be retrieved and deleted
  43  * with the following line of code. The variable se is a
  44  * SOAPEnvelope object.
  45  * 

  46  *      se.getHeader().detachNode();
  47  * 
  48  *
  49  * A SOAPHeader object is created with the SOAPEnvelope
  50  * method addHeader. This method, which creates a new header and adds it
  51  * to the envelope, may be called only after the existing header has been removed.
  52  *
  53  * 

  54  *      se.getHeader().detachNode();
  55  *      SOAPHeader sh = se.addHeader();
  56  * 
  57  * 

  58  * A SOAPHeader object can have only SOAPHeaderElement
  59  * objects as its immediate children. The method addHeaderElement
  60  * creates a new HeaderElement object and adds it to the
  61  * SOAPHeader object. In the following line of code, the
  62  * argument to the method addHeaderElement is a Name
  63  * object that is the name for the new HeaderElement object.
  64  * 

  65  *      SOAPHeaderElement shElement = sh.addHeaderElement(name);
  66  * 
  67  *
  68  * @see SOAPHeaderElement
  69  */
  70 public interface SOAPHeader extends SOAPElement {
  71     /**
  72      * Creates a new SOAPHeaderElement object initialized with the
  73      * specified name and adds it to this SOAPHeader object.
  74      *
  75      * @param name a Name object with the name of the new
  76      *        SOAPHeaderElement object
  77      * @return the new SOAPHeaderElement object that was
  78      *          inserted into this SOAPHeader object
  79      * @exception SOAPException if a SOAP error occurs
  80      * @see SOAPHeader#addHeaderElement(javax.xml.namespace.QName)
  81      */
  82     public SOAPHeaderElement addHeaderElement(Name name)
  83         throws SOAPException;
  84 
  85     /**
  86      * Creates a new SOAPHeaderElement object initialized with the
  87      * specified qname and adds it to this SOAPHeader object.

  89      * @param qname a QName object with the qname of the new
  90      *        SOAPHeaderElement object
  91      * @return the new SOAPHeaderElement object that was
  92      *          inserted into this SOAPHeader object
  93      * @exception SOAPException if a SOAP error occurs
  94      * @see SOAPHeader#addHeaderElement(Name)
  95      * @since SAAJ 1.3
  96      */
  97     public SOAPHeaderElement addHeaderElement(QName qname)
  98         throws SOAPException;

 100     /**
 101      * Returns an Iterator over all the SOAPHeaderElement objects
 102      * in this SOAPHeader object
 103      * that have the specified actor and that have a MustUnderstand attribute
 104      * whose value is equivalent to true.
 105      * 
 106      * In SOAP 1.2 the env:actor attribute is replaced by the env:role
 107      * attribute, but with essentially the same semantics.
 108      *
 109      * @param actor a String giving the URI of the actor / role
 110      *        for which to search
 111      * @return an Iterator object over all the
 112      *         SOAPHeaderElement objects that contain the specified
 113      *          actor / role and are marked as MustUnderstand
 114      * @see #examineHeaderElements
 115      * @see #extractHeaderElements
 116      * @see SOAPConstants#URI_SOAP_ACTOR_NEXT
 117      *
 118      * @since SAAJ 1.2
 119      */
 120     public Iterator examineMustUnderstandHeaderElements(String actor);
 121 
 122     /**
 123      * Returns an Iterator over all the SOAPHeaderElement objects
 124      * in this SOAPHeader object
 125      * that have the specified actor.
 126      *
 127      * An actor is a global attribute that indicates the intermediate
 128      * parties that should process a message before it reaches its ultimate
 129      * receiver. An actor receives the message and processes it before sending
 130      * it on to the next actor. The default actor is the ultimate intended
 131      * recipient for the message, so if no actor attribute is included in a
 132      * SOAPHeader object, it is sent to the ultimate receiver
 133      * along with the message body.
 134      * 
 135      * In SOAP 1.2 the env:actor attribute is replaced by the env:role
 136      * attribute, but with essentially the same semantics.
 137      *
 138      * @param actor a String giving the URI of the actor / role
 139      *        for which to search
 140      * @return an Iterator object over all the
 141      *         SOAPHeaderElement objects that contain the specified
 142      *          actor / role
 143      * @see #extractHeaderElements
 144      * @see SOAPConstants#URI_SOAP_ACTOR_NEXT
 145      */
 146     public Iterator examineHeaderElements(String actor);
 147

 152      * from this SOAPHeader object.

 154      * This method allows an actor to process the parts of the
 155      * SOAPHeader object that apply to it and to remove
 156      * them before passing the message on to the next actor.
 157      * 
 158      * In SOAP 1.2 the env:actor attribute is replaced by the env:role

 163      * @return an Iterator object over all the
 164      *         SOAPHeaderElement objects that contain the specified
 165      *          actor / role
 166      *
 167      * @see #examineHeaderElements

 170     public Iterator extractHeaderElements(String actor);
