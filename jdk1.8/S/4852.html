  38 /**
  39  *
  40  *
  41  * The XPathNamespace interface is returned by
  42  * XPathResult interfaces to represent the XPath namespace node
  43  * type that DOM lacks. There is no public constructor for this node type.
  44  * Attempts to place it into a hierarchy or a NamedNodeMap result in a
  45  * DOMException with the code HIERARCHY_REQUEST_ERR
  46  * . This node is read only, so methods or setting of attributes that would
  47  * mutate the node result in a DOMException with the code
  48  * NO_MODIFICATION_ALLOWED_ERR.
  49  * The core specification describes attributes of the Node
  50  * interface that are different for different node node types but does not
  51  * describe XPATH_NAMESPACE_NODE, so here is a description of
  52  * those attributes for this node type. All attributes of Node
  53  * not described in this section have a null or
  54  * false value.
  55  * ownerDocument matches the ownerDocument of the
  56  * ownerElement even if the element is later adopted.
  57  * prefix is the prefix of the namespace represented by the
  58  * node.
  59  * nodeName is the same as prefix.
  60  * nodeType is equal to XPATH_NAMESPACE_NODE.
  61  * namespaceURI is the namespace URI of the namespace
  62  * represented by the node.
  63  * adoptNode, cloneNode, and
  64  * importNode fail on this node type by raising a
  65  * DOMException with the code NOT_SUPPORTED_ERR.In
  66  * future versions of the XPath specification, the definition of a namespace
  67  * node may be changed incomatibly, in which case incompatible changes to
  68  * field values may be required to implement versions beyond XPath 1.0.
  69  * See also the Document Object Model (DOM) Level 3 XPath Specification.
  70  *
  71  * This implementation wraps the DOM attribute node that contained the
  72  * namespace declaration.
  73  * @xsl.usage internal
  74  */
  75
