  28 /**
  29  * The Dictionary class is the abstract parent of any
  30  * class, such as Hashtable, which maps keys to values.
  31  * Every key and every value is an object. In any one Dictionary
  32  * object, every key is associated with at most one value. Given a
  33  * Dictionary and a key, the associated element can be looked up.
  34  * Any non-null object can be used as a key and as a value.
  35  * 
  36  * As a rule, the equals method should be used by
  37  * implementations of this class to decide if two keys are the same.
  38  * 
  39  * NOTE: This class is obsolete.  New implementations should
  40  * implement the Map interface, rather than extending this class.
  41  *
  42  * @author  unascribed
  43  * @see     java.util.Map
  44  * @see     java.lang.Object#equals(java.lang.Object)
  45  * @see     java.lang.Object#hashCode()
  46  * @see     java.util.Hashtable
  47  * @since   JDK1.0
  48  */

 107      *          null if the key is not mapped to any value in

 109      * @exception NullPointerException if the key is null.
 110      * @see     java.util.Dictionary#put(java.lang.Object, java.lang.Object)
 111      */
 112     abstract public V get(Object key);

 114     /**
 115      * Maps the specified key to the specified
 116      * value in this dictionary. Neither the key nor the
 117      * value can be null.
 118      * 
 119      * If this dictionary already contains an entry for the specified
 120      * key, the value already in this dictionary for that

 126      *
