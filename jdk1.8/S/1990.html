  30 /**
  31  * A container for data of a specific primitive type.
  32  *
  33  *  A buffer is a linear, finite sequence of elements of a specific
  34  * primitive type.  Aside from its content, the essential properties of a
  35  * buffer are its capacity, limit, and position: 
  36  *
  37  * 

  38  *
  39  *    A buffer's capacity is the number of elements it contains.  The
  40  *   capacity of a buffer is never negative and never changes.  
  41  *
  42  *    A buffer's limit is the index of the first element that should
  43  *   not be read or written.  A buffer's limit is never negative and is never
  44  *   greater than its capacity.  
  45  *
  46  *    A buffer's position is the index of the next element to be
  47  *   read or written.  A buffer's position is never negative and is never
  48  *   greater than its limit.  
  49  *
  50  * 
  51  *
  52  *  There is one subclass of this class for each non-boolean primitive type.
  53  *
  54  *
  55  * 
 Transferring data 

  56  *
  57  *  Each subclass of this class defines two categories of get and
  58  * put operations: 
  59  *
  60  * 

  61  *
  62  *    Relative operations read or write one or more elements starting
  63  *   at the current position and then increment the position by the number of
  64  *   elements transferred.  If the requested transfer exceeds the limit then a
  65  *   relative get operation throws a {@link BufferUnderflowException}
  66  *   and a relative put operation throws a {@link
  67  *   BufferOverflowException}; in either case, no data is transferred.  
  68  *
  69  *    Absolute operations take an explicit element index and do not
  70  *   affect the position.  Absolute get and put operations throw
  71  *   an {@link IndexOutOfBoundsException} if the index argument exceeds the
  72  *   limit.  
  73  *
  74  * 
  75  *
  76  *  Data may also, of course, be transferred in to or out of a buffer by the
  77  * I/O operations of an appropriate channel, which are always relative to the
  78  * current position.
  79  *
  80  *
  81  * 
 Marking and resetting 

  82  *
  83  *  A buffer's mark is the index to which its position will be reset
  84  * when the {@link #reset reset} method is invoked.  The mark is not always
  85  * defined, but when it is defined it is never negative and is never greater
  86  * than the position.  If the mark is defined then it is discarded when the
  87  * position or the limit is adjusted to a value smaller than the mark.  If the
  88  * mark is not defined then invoking the {@link #reset reset} method causes an
  89  * {@link InvalidMarkException} to be thrown.
  90  *
  91  *
  92  * 
 Invariants 

  93  *
  94  *  The following invariant holds for the mark, position, limit, and
  95  * capacity values:
  96  *
  97  * 

  98  *     0 <=
  99  *     mark <=
 100  *     position <=
 101  *     limit <=
 102  *     capacity
 103  * 
 104  *
 105  *  A newly-created buffer always has a position of zero and a mark that is
 106  * undefined.  The initial limit may be zero, or it may be some other value
 107  * that depends upon the type of the buffer and the manner in which it is
 108  * constructed.  Each element of a newly-allocated buffer is initialized
 109  * to zero.
 110  *
 111  *
 112  * 
 Clearing, flipping, and rewinding 

 113  *
 114  *  In addition to methods for accessing the position, limit, and capacity
 115  * values and for marking and resetting, this class also defines the following
 116  * operations upon buffers:
 117  *
 118  * 
    
 119  *
 120  *   
  •  {@link #clear} makes a buffer ready for a new sequence of
    • 
 121  *   channel-read or relative put operations: It sets the limit to the
 122  *   capacity and the position to zero.  
 123  *
 124  *   
  •  {@link #flip} makes a buffer ready for a new sequence of
    • 
 125  *   channel-write or relative get operations: It sets the limit to the
 126  *   current position and then sets the position to zero.  
 127  *
 128  *   
  •  {@link #rewind} makes a buffer ready for re-reading the data that
    • 
 129  *   it already contains: It leaves the limit unchanged and sets the position
 130  *   to zero.  
 131  *
 132  * 
 133  *
 134  *
 135  * 
     Read-only buffers 
    
 136  *
 137  *  Every buffer is readable, but not every buffer is writable.  The
 138  * mutation methods of each buffer class are specified as optional
 139  * operations that will throw a {@link ReadOnlyBufferException} when
 140  * invoked upon a read-only buffer.  A read-only buffer does not allow its
 141  * content to be changed, but its mark, position, and limit values are mutable.
 142  * Whether or not a buffer is read-only may be determined by invoking its
 143  * {@link #isReadOnly isReadOnly} method.
 144  *
 145  *
 146  * 
     Thread safety 
    
 147  *
 148  *  Buffers are not safe for use by multiple concurrent threads.  If a
 149  * buffer is to be used by more than one thread then access to the buffer
 150  * should be controlled by appropriate synchronization.
 151  *
 152  *
 153  * 
     Invocation chaining 
    
 154  *
 155  *  Methods in this class that do not otherwise have a value to return are
 156  * specified to return the buffer upon which they are invoked.  This allows
 157  * method invocations to be chained; for example, the sequence of statements
 158  *
 159  * 
    
 160  * b.flip();
 161  * b.position(23);
 162  * b.limit(42);
 163  *
 164  * can be replaced by the single, more compact statement
 165  *
 166  * 
    
 167  * b.flip().position(23).limit(42);
 168  *
 169  *
 170  * @author Mark Reinhold
 171  * @author JSR-51 Expert Group
 172  * @since 1.4
 173  */
 174

 194     // Creates a new buffer with the given mark, position, limit, and capacity,
 195     // after checking invariants.
 196     //
 197     Buffer(int mark, int pos, int lim, int cap) {       // package-private
 198         if (cap < 0)
 199             throw new IllegalArgumentException("Negative capacity: " + cap);
 200         this.capacity = cap;
 201         limit(lim);
 202         position(pos);
 203         if (mark >= 0) {
 204             if (mark > pos)
 205                 throw new IllegalArgumentException("mark > position: ("
 206                                                    + mark + " > " + pos + ")");
 207             this.mark = mark;
 208         }
 209     }
