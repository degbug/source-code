  40 /**
  41  * The "viewport" or "porthole" through which you see the underlying
  42  * information. When you scroll, what moves is the viewport. It is like
  43  * peering through a camera's viewfinder. Moving the viewfinder upwards
  44  * brings new things into view at the top of the picture and loses
  45  * things that were at the bottom.
  46  * 
  47  * By default, JViewport is opaque. To change this, use the
  48  * setOpaque method.
  49  * 
  50  * NOTE:We have implemented a faster scrolling algorithm that
  51  * does not require a buffer to draw in. The algorithm works as follows:
  52  * 
  1. The view and parent view and checked to see if they are

  53  * JComponents,
  54  * if they aren't, stop and repaint the whole viewport.
  55  * 
  • If the viewport is obscured by an ancestor, stop and repaint the whole
    • 
  56  * viewport.
  57  * 
  • Compute the region that will become visible, if it is as big as
    • 
  58  * the viewport, stop and repaint the whole view region.
  59  * 
  • Obtain the ancestor Window's graphics and
    • 
  60  * do a copyArea on the scrolled region.
  61  * 
  • Message the view to repaint the newly visible region.
    • 
  62  * 
  • The next time paint is invoked on the viewport, if the clip region
    • 
  63  * is smaller than the viewport size a timer is kicked off to repaint the
  64  * whole region.
  65  * 
  66  * In general this approach is much faster. Compared to the backing store
  67  * approach this avoids the overhead of maintaining an offscreen buffer and
  68  * having to do two copyAreas.
  69  * Compared to the non backing store case this
  70  * approach will greatly reduce the painted region.
  71  * 
  72  * This approach can cause slower times than the backing store approach
  73  * when the viewport is obscured by another window, or partially offscreen.
  74  * When another window
  75  * obscures the viewport the copyArea will copy garbage and a
  76  * paint event will be generated by the system to inform us we need to
  77  * paint the newly exposed region. The only way to handle this is to
  78  * repaint the whole viewport, which can cause slower performance than the
  79  * backing store case. In most applications very rarely will the user be
  80  * scrolling while the viewport is obscured by another window or offscreen,
  81  * so this optimization is usually worth the performance hit when obscured.
  82  * 
  83  * Warning: Swing is not thread safe. For more
  84  * information see 
  85  * href="package-summary.html#threading">Swing's Threading
  86  * Policy.
  87  * 
  88  * Warning:
  89  * Serialized objects of this class will not be compatible with
  90  * future Swing releases. The current serialization support is
  91  * appropriate for short term storage or RMI between applications running
  92  * the same version of Swing.  As of 1.4, support for long term storage
  93  * of all JavaBeansâ„¢
  94  * has been added to the java.beans package.
  95  * Please see {@link java.beans.XMLEncoder}.

 208     //
 209     // Window blitting:
 210     //
 211     // As mentioned in the javadoc when using windowBlit a paint event
 212     // will be generated by the system if copyArea copies a non-visible
 213     // portion of the view (in other words, it copies garbage). We are
 214     // not guaranteed to receive the paint event before other mouse events,
 215     // so we can not be sure we haven't already copied garbage a bunch of
 216     // times to different parts of the view. For that reason when a blit
 217     // happens and the Component is obscured (the check for obscurity
 218     // is not supported on all platforms and is checked via ComponentPeer
 219     // methods) the ivar repaintAll is set to true. When paint is received
 220     // if repaintAll is true (we previously did a blit) it is set to
 221     // false, and if the clip region is smaller than the viewport
 222     // waitingForRepaint is set to true and a timer is started. When
 223     // the timer fires if waitingForRepaint is true, repaint is invoked.
 224     // In the mean time, if the view is asked to scroll and waitingForRepaint
 225     // is true, a blit will not happen, instead the non-backing store case
 226     // of scrolling will happen, which will reset waitingForRepaint.
 227     // waitingForRepaint is set to false in paint when the clip rect is
 228     // bigger (or equal) to the size of the viewport.
 229     // A Timer is used instead of just a repaint as it appeared to offer
 230     // better performance.

 362     /**
 363      * Scrolls the view so that Rectangle
 364      * within the view becomes visible.
 365      * 
 366      * This attempts to validate the view before scrolling if the
 367      * view is currently not valid - isValid returns false.
 368      * To avoid excessive validation when the containment hierarchy is
 369      * being created this will not validate if one of the ancestors does not
 370      * have a peer, or there is no validate root ancestor, or one of the
 371      * ancestors is not a Window or Applet.
 372      * 
 373      * Note that this method will not scroll outside of the
 374      * valid viewport; for example, if contentRect is larger
 375      * than the viewport, scrolling will be confined to the viewport's
 376      * bounds.
 377      *
 378      * @param contentRect the Rectangle to display
 379      * @see JComponent#isValidateRoot
 380      * @see java.awt.Component#isValid
 381      * @see java.awt.Component#getPeer
 382      */
 383     public void scrollRectToVisible(Rectangle contentRect) {
 384         Component view = getView();
 385 
 386         if (view == null) {
 387             return;
 388         } else {
 389             if (!view.isValid()) {
 390                 // If the view is not valid, validate. scrollRectToVisible
 391                 // may fail if the view is not valid first, contentRect
 392                 // could be bigger than invalid size.
 393                 validateView();
 394             }
 395             int dx, dy;
 396 
 397             dx = positionAdjustment(getWidth(), contentRect.width, contentRect.x);
 398             dy = positionAdjustment(getHeight(), contentRect.height, contentRect.y);
 399 
 400             if (dx != 0 || dy != 0) {
 401                 Point viewPosition = getViewPosition();
 402                 Dimension viewSize = view.getSize();
 403                 int startX = viewPosition.x;
 404                 int startY = viewPosition.y;
 405                 Dimension extent = getExtentSize();
 406 
 407                 viewPosition.x -= dx;
 408                 viewPosition.y -= dy;
 409                 // Only constrain the location if the view is valid. If the
 410                 // the view isn't valid, it typically indicates the view
 411                 // isn't visible yet and most likely has a bogus size as will
 412                 // we, and therefore we shouldn't constrain the scrolling
 413                 if (view.isValid()) {
 414                     if (getParent().getComponentOrientation().isLeftToRight()) {
 415                         if (viewPosition.x + extent.width > viewSize.width) {
 416                             viewPosition.x = Math.max(0, viewSize.width - extent.width);
 417                         } else if (viewPosition.x < 0) {
 418                             viewPosition.x = 0;
 419                         }
 420                     } else {
 421                         if (extent.width > viewSize.width) {
 422                             viewPosition.x = viewSize.width - extent.width;
 423                         } else {
 424                             viewPosition.x = Math.max(0, Math.min(viewSize.width - extent.width, viewPosition.x));
 425                         }
 426                     }
 427                     if (viewPosition.y + extent.height > viewSize.height) {
 428                         viewPosition.y = Math.max(0, viewSize.height -
 429                                                   extent.height);
 430                     }
 431                     else if (viewPosition.y < 0) {
 432                         viewPosition.y = 0;
 433                     }
 434                 }
 435                 if (viewPosition.x != startX || viewPosition.y != startY) {
 436                     setViewPosition(viewPosition);
 437                     // NOTE: How JViewport currently works with the
 438                     // backing store is not foolproof. The sequence of
 439                     // events when setViewPosition
 440                     // (scrollRectToVisible) is called is to reset the
 441                     // views bounds, which causes a repaint on the
 442                     // visible region and sets an ivar indicating
 443                     // scrolling (scrollUnderway). When
 444                     // JViewport.paint is invoked if scrollUnderway is
 445                     // true, the backing store is blitted.  This fails
 446                     // if between the time setViewPosition is invoked
 447                     // and paint is received another repaint is queued
 448                     // indicating part of the view is invalid. There
 449                     // is no way for JViewport to notice another
 450                     // repaint has occurred and it ends up blitting
 451                     // what is now a dirty region and the repaint is
 452                     // never delivered.
 453                     // It just so happens JTable encounters this
 454                     // behavior by way of scrollRectToVisible, for
 455                     // this reason scrollUnderway is set to false
 456                     // here, which effectively disables the backing
 457                     // store.
 458                     scrollUnderway = false;
 459                 }
 460             }
 461         }
 462     }

 465      * Ascends the Viewport's parents stopping when
 466      * a component is found that returns
 467      * true to isValidateRoot.
 468      * If all the Component's  parents are visible,
 469      * validate will then be invoked on it. The

 493      /*  Used by the scrollRectToVisible method to determine the
 494       *  proper direction and amount to move by. The integer variables are named
 495       *  width, but this method is applicable to height also. The code assumes that
 496       *  parentWidth/childWidth are positive and childAt can be negative.
 497       */
 498     private int positionAdjustment(int parentWidth, int childWidth, int childAt)    {
 499 
 500         //   +-----+
 501         //   | --- |     No Change
 502         //   +-----+
 503         if (childAt >= 0 && childWidth + childAt <= parentWidth)    {
 504             return 0;
 505         }
 506 
 507         //   +-----+
 508         //  ---------   No Change
 509         //   +-----+
 510         if (childAt <= 0 && childWidth + childAt >= parentWidth) {
 511             return 0;
 512         }
 513 
 514         //   +-----+          +-----+
 515         //   |   ----    ->   | ----|
 516         //   +-----+          +-----+
 517         if (childAt > 0 && childWidth <= parentWidth)    {
 518             return -childAt + parentWidth - childWidth;
 519         }
 520 
 521         //   +-----+             +-----+
 522         //   |  --------  ->     |--------
 523         //   +-----+             +-----+
 524         if (childAt >= 0 && childWidth >= parentWidth)   {
 525             return -childAt;
 526         }
 527 
 528         //   +-----+          +-----+
 529         // ----    |     ->   |---- |
 530         //   +-----+          +-----+
 531         if (childAt <= 0 && childWidth <= parentWidth)   {
 532             return -childAt;
 533         }
 534 
 535         //   +-----+             +-----+
 536         //-------- |      ->   --------|
 537         //   +-----+             +-----+
 538         if (childAt < 0 && childWidth >= parentWidth)    {
 539             return -childAt + parentWidth - childWidth;
 540         }
 541 
 542         return 0;
 543     }

 680      */
 681     public void paint(Graphics g)
 682     {
 683         int width = getWidth();
 684         int height = getHeight();
 685 
 686         if ((width <= 0) || (height <= 0)) {
 687             return;
 688         }
 689 
 690         if (inBlitPaint) {
 691             // We invoked paint as part of copyArea cleanup, let it through.
 692             super.paint(g);
 693             return;
 694         }
 695 
 696         if (repaintAll) {
 697             repaintAll = false;
 698             Rectangle clipB = g.getClipBounds();
 699             if (clipB.width < getWidth() ||
 700                 clipB.height < getHeight()) {
 701                 waitingForRepaint = true;
 702                 if (repaintTimer == null) {
 703                     repaintTimer = createRepaintTimer();
 704                 }
 705                 repaintTimer.stop();
 706                 repaintTimer.start();
 707                 // We really don't need to paint, a future repaint will
 708                 // take care of it, but if we don't we get an ugly flicker.
 709             }
 710             else {
 711                 if (repaintTimer != null) {
 712                     repaintTimer.stop();
 713                 }
 714                 waitingForRepaint = false;
 715             }
 716         }
 717         else if (waitingForRepaint) {
 718             // Need a complete repaint before resetting waitingForRepaint
 719             Rectangle clipB = g.getClipBounds();
 720             if (clipB.width >= getWidth() &&
 721                 clipB.height >= getHeight()) {
 722                 waitingForRepaint = false;
 723                 repaintTimer.stop();
 724             }
 725         }
 726 
 727         if (!backingStore || isBlitting() || getView() == null) {
 728             super.paint(g);
 729             lastPaintPosition = getViewLocation();
 730             return;
 731         }
 732 
 733         // If the view is smaller than the viewport and we are not opaque
 734         // (that is, we won't paint our background), we should set the
 735         // clip. Otherwise, as the bounds of the view vary, we will
 736         // blit garbage into the exposed areas.
 737         Rectangle viewBounds = getView().getBounds();
 738         if (!isOpaque()) {
 739             g.clipRect(0, 0, viewBounds.width, viewBounds.height);
 740         }
 741 
 742         if (backingStoreImage == null) {
 743             // Backing store is enabled but this is the first call to paint.
 744             // Create the backing store, paint it and then copy to g.
 745             // The backing store image will be created with the size of
 746             // the viewport. We must make sure the clip region is the
 747             // same size, otherwise when scrolling the backing image
 748             // the region outside of the clipped region will not be painted,
 749             // and result in empty areas.
 750             backingStoreImage = createImage(width, height);
 751             Rectangle clip = g.getClipBounds();
 752             if (clip.width != width || clip.height != height) {
 753                 if (!isOpaque()) {
 754                     g.setClip(0, 0, Math.min(viewBounds.width, width),
 755                               Math.min(viewBounds.height, height));
 756                 }
 757                 else {
 758                     g.setClip(0, 0, width, height);
 759                 }
 760                 paintViaBackingStore(g, clip);
 761             }
 762             else {
 763                 paintViaBackingStore(g);
 764             }
 765         }
 766         else {
 767             if (!scrollUnderway || lastPaintPosition.equals(getViewLocation())) {
 768                 // No scrolling happened: repaint required area via backing store.
 769                 paintViaBackingStore(g);
 770             } else {
 771                 // The image was scrolled. Manipulate the backing store and flush it to g.
 772                 Point blitFrom = new Point();
 773                 Point blitTo = new Point();
 774                 Dimension blitSize = new Dimension();
 775                 Rectangle blitPaint = new Rectangle();
 776 
 777                 Point newLocation = getViewLocation();
 778                 int dx = newLocation.x - lastPaintPosition.x;
 779                 int dy = newLocation.y - lastPaintPosition.y;
 780                 boolean canBlit = computeBlit(dx, dy, blitFrom, blitTo, blitSize, blitPaint);
 781                 if (!canBlit) {
 782                     // The image was either moved diagonally or
 783                     // moved by more than the image size: paint normally.
 784                     paintViaBackingStore(g);
 785                 } else {
 786                     int bdx = blitTo.x - blitFrom.x;
 787                     int bdy = blitTo.y - blitFrom.y;
 788 
 789                     // Move the relevant part of the backing store.
 790                     Rectangle clip = g.getClipBounds();
 791                     // We don't want to inherit the clip region when copying
 792                     // bits, if it is inherited it will result in not moving
 793                     // all of the image resulting in garbage appearing on
 794                     // the screen.
 795                     g.setClip(0, 0, width, height);
 796                     Graphics bsg = getBackingStoreGraphics(g);
 797                     try {
 798                         bsg.copyArea(blitFrom.x, blitFrom.y, blitSize.width, blitSize.height, bdx, bdy);
 799 
 800                         g.setClip(clip.x, clip.y, clip.width, clip.height);
 801                         // Paint the rest of the view; the part that has just been exposed.
 802                         Rectangle r = viewBounds.intersection(blitPaint);
 803                         bsg.setClip(r);
 804                         super.paint(bsg);
 805 
 806                         // Copy whole of the backing store to g.
 807                         g.drawImage(backingStoreImage, 0, 0, this);
 808                     } finally {
 809                         bsg.dispose();
 810                     }
 811                 }
 812             }
 813         }
 814         lastPaintPosition = getViewLocation();
 815         scrollUnderway = false;
 816     }

1047      * Sets the view coordinates that appear in the upper left
1048      * hand corner of the viewport, does nothing if there's no view.
1049      *
1050      * @param p  a Point object giving the upper left coordinates
1051      */
1052     public void setViewPosition(Point p)
1053     {
1054         Component view = getView();
1055         if (view == null) {
1056             return;
1057         }
1058 
1059         int oldX, oldY, x = p.x, y = p.y;
1060 
1061         /* Collect the old x,y values for the views location
1062          * and do the song and dance to avoid allocating
1063          * a Rectangle object if we don't have to.
1064          */
1065         if (view instanceof JComponent) {
1066             JComponent c = (JComponent)view;
1067             oldX = c.getX();
1068             oldY = c.getY();
1069         }
1070         else {
1071             Rectangle r = view.getBounds();
1072             oldX = r.x;
1073             oldY = r.y;
1074         }
1075 
1076         /* The view scrolls in the opposite direction to mouse
1077          * movement.
1078          */
1079         int newX = -x;
1080         int newY = -y;
1081 
1082         if ((oldX != newX) || (oldY != newY)) {
1083             if (!waitingForRepaint && isBlitting() && canUseWindowBlitter()) {
1084                 RepaintManager rm = RepaintManager.currentManager(this);
1085                 // The cast to JComponent will work, if view is not
1086                 // a JComponent, isBlitting will return false.
1087                 JComponent jview = (JComponent)view;
1088                 Rectangle dirty = rm.getDirtyRegion(jview);
1089                 if (dirty == null || !dirty.contains(jview.getVisibleRect())) {
1090                     rm.beginPaint();
1091                     try {
1092                         Graphics g = JComponent.safelyGetGraphics(this);
1093                         flushViewDirtyRegion(g, dirty);
1094                         view.setLocation(newX, newY);
1095                         Rectangle r = new Rectangle(
1096                             0, 0, getWidth(), Math.min(getHeight(), jview.getHeight()));
1097                         g.setClip(r);
1098                         // Repaint the complete component if the blit succeeded
1099                         // and needsRepaintAfterBlit returns true.
1100                         repaintAll = (windowBlitPaint(g) &&
1101                                       needsRepaintAfterBlit());
1102                         g.dispose();
1103                         rm.notifyRepaintPerformed(this, r.x, r.y, r.width, r.height);
1104                         rm.markCompletelyClean((JComponent)getParent());
1105                         rm.markCompletelyClean(this);
1106                         rm.markCompletelyClean(jview);
1107                     } finally {
1108                         rm.endPaint();
1109                     }
1110                 }
1111                 else {
1112                     // The visible region is dirty, no point in doing copyArea
1113                     view.setLocation(newX, newY);
1114                     repaintAll = false;
1115                 }
1116             }
1117             else {
1118                 scrollUnderway = true;
1119                 // This calls setBounds(), and then repaint().
1120                 view.setLocation(newX, newY);
1121                 repaintAll = false;
1122             }
1123             // we must validate the hierarchy to not break the hw/lw mixing
1124             revalidate();
1125             fireStateChanged();
1126         }
1127     }

1143     /**
1144      * Computes the parameters for a blit where the backing store image
1145      * currently contains oldLoc in the upper left hand corner
1146      * and we're scrolling to newLoc.
1147      * The parameters are modified
1148      * to return the values required for the blit.
1149      *
1150      * @param dx  the horizontal delta
1151      * @param dy  the vertical delta
1152      * @param blitFrom the Point we're blitting from
1153      * @param blitTo the Point we're blitting to
1154      * @param blitSize the Dimension of the area to blit
1155      * @param blitPaint the area to blit
1156      * @return  true if the parameters are modified and we're ready to blit;
1157      *          false otherwise
1158      */
1159     protected boolean computeBlit(
1160         int dx,
1161         int dy,
1162         Point blitFrom,
1163         Point blitTo,
1164         Dimension blitSize,
1165         Rectangle blitPaint)
1166     {
1167         int dxAbs = Math.abs(dx);
1168         int dyAbs = Math.abs(dy);
1169         Dimension extentSize = getExtentSize();
1170 
1171         if ((dx == 0) && (dy != 0) && (dyAbs < extentSize.height)) {
1172             if (dy < 0) {
1173                 blitFrom.y = -dy;
1174                 blitTo.y = 0;
1175                 blitPaint.y = extentSize.height + dy;
1176             }
1177             else {
1178                 blitFrom.y = 0;
1179                 blitTo.y = dy;
1180                 blitPaint.y = 0;
1181             }
1182 
1183             blitPaint.x = blitFrom.x = blitTo.x = 0;
1184 
1185             blitSize.width = extentSize.width;
1186             blitSize.height = extentSize.height - dyAbs;
1187 
1188             blitPaint.width = extentSize.width;
1189             blitPaint.height = dyAbs;
1190 
1191             return true;
1192         }
1193 
1194         else if ((dy == 0) && (dx != 0) && (dxAbs < extentSize.width)) {
1195             if (dx < 0) {
1196                 blitFrom.x = -dx;
1197                 blitTo.x = 0;
1198                 blitPaint.x = extentSize.width + dx;
1199             }
1200             else {
1201                 blitFrom.x = 0;
1202                 blitTo.x = dx;
1203                 blitPaint.x = 0;
1204             }
1205 
1206             blitPaint.y = blitFrom.y = blitTo.y = 0;
1207 
1208             blitSize.width = extentSize.width - dxAbs;
1209             blitSize.height = extentSize.height;
1210 
1211             blitPaint.width = dxAbs;
1212             blitPaint.height = extentSize.height;
1213 
1214             return true;
1215         }
1216 
1217         else {
1218             return false;
1219         }
1220     }

1443     /**
1444      * Returns true if the component needs to be completely repainted after
1445      * a blit and a paint is received.
1446      */
1447     private boolean needsRepaintAfterBlit() {
1448         // Find the first heavy weight ancestor. isObscured and
1449         // canDetermineObscurity are only appropriate for heavy weights.
1450         Component heavyParent = getParent();
1451 
1452         while (heavyParent != null && heavyParent.isLightweight()) {
1453             heavyParent = heavyParent.getParent();
1454         }
1455 
1456         if (heavyParent != null) {
1457             ComponentPeer peer = heavyParent.getPeer();
1458 
1459             if (peer != null && peer.canDetermineObscurity() &&
1460                                 !peer.isObscured()) {
1461                 // The peer says we aren't obscured, therefore we can assume
1462                 // that we won't later be messaged to paint a portion that
1463                 // we tried to blit that wasn't valid.
1464                 // It is certainly possible that when we blited we were
1465                 // obscured, and by the time this is invoked we aren't, but the
1466                 // chances of that happening are pretty slim.
1467                 return false;
1468             }
1469         }
1470         return true;

1513     /**
1514      * Used when blitting.
1515      *
1516      * @param g  the Graphics context within which to paint
1517      * @return true if blitting succeeded; otherwise false
1518      */
1519     private boolean windowBlitPaint(Graphics g) {
1520         int width = getWidth();
1521         int height = getHeight();
1522 
1523         if ((width == 0) || (height == 0)) {
1524             return false;
1525         }
1526 
1527         boolean retValue;
1528         RepaintManager rm = RepaintManager.currentManager(this);
1529         JComponent view = (JComponent) getView();
1530 
1531         if (lastPaintPosition == null ||
1532             lastPaintPosition.equals(getViewLocation())) {
1533             paintView(g);
1534             retValue = false;
1535         } else {
1536             // The image was scrolled. Manipulate the backing store and flush
1537             // it to g.
1538             Point blitFrom = new Point();
1539             Point blitTo = new Point();
1540             Dimension blitSize = new Dimension();
1541             Rectangle blitPaint = new Rectangle();
1542 
1543             Point newLocation = getViewLocation();
1544             int dx = newLocation.x - lastPaintPosition.x;
1545             int dy = newLocation.y - lastPaintPosition.y;
1546             boolean canBlit = computeBlit(dx, dy, blitFrom, blitTo, blitSize,
1547                                           blitPaint);
1548             if (!canBlit) {
1549                 paintView(g);
1550                 retValue = false;
1551             } else {
1552                 // Prepare the rest of the view; the part that has just been
1553                 // exposed.
1554                 Rectangle r = view.getBounds().intersection(blitPaint);
1555                 r.x -= view.getX();
1556                 r.y -= view.getY();
1557 
1558                 blitDoubleBuffered(view, g, r.x, r.y, r.width, r.height,
1559                                    blitFrom.x, blitFrom.y, blitTo.x, blitTo.y,
1560                                    blitSize.width, blitSize.height);
1561                 retValue = true;
1562             }
1563         }
1564         lastPaintPosition = getViewLocation();
1565         return retValue;
1566     }

1568     //
1569     // NOTE: the code below uses paintForceDoubleBuffered for historical
1570     // reasons.  If we're going to allow a blit we've already accounted for
1571     // everything that paintImmediately and _paintImmediately does, for that
1572     // reason we call into paintForceDoubleBuffered to diregard whether or
1573     // not setDoubleBuffered(true) was invoked on the view.
1574     //
1575 
1576     private void blitDoubleBuffered(JComponent view, Graphics g,
1577                                     int clipX, int clipY, int clipW, int clipH,
1578                                     int blitFromX, int blitFromY, int blitToX, int blitToY,
1579                                     int blitW, int blitH) {
1580         // NOTE:
1581         //   blitFrom/blitTo are in JViewport coordinates system
1582         //     not the views coordinate space.
1583         //   clip* are in the views coordinate space.
1584         RepaintManager rm = RepaintManager.currentManager(this);
1585         int bdx = blitToX - blitFromX;
1586         int bdy = blitToY - blitFromY;
1587 
1588         Composite oldComposite = null;
1589         // Shift the scrolled region
1590         if (g instanceof Graphics2D) {
1591             Graphics2D g2d = (Graphics2D) g;
1592             oldComposite = g2d.getComposite();
1593             g2d.setComposite(AlphaComposite.Src);
1594         }
1595         rm.copyArea(this, g, blitFromX, blitFromY, blitW, blitH, bdx, bdy,
1596                     false);
1597         if (oldComposite != null) {
1598             ((Graphics2D) g).setComposite(oldComposite);
1599         }
1600         // Paint the newly exposed region.
1601         int x = view.getX();
1602         int y = view.getY();
1603         g.translate(x, y);
1604         g.setClip(clipX, clipY, clipW, clipH);
1605         view.paintForceDoubleBuffered(g);
1606         g.translate(-x, -y);
1607     }

1609     /**
1610      * Called to paint the view, usually when blitPaint
1611      * can not blit.
1612      *
1613      * @param g the Graphics context within which to paint
1614      */
1615     private void paintView(Graphics g) {
1616         Rectangle clip = g.getClipBounds();
1617         JComponent view = (JComponent)getView();
1618 
1619         if (view.getWidth() >= getWidth()) {
1620             // Graphics is relative to JViewport, need to map to view's
1621             // coordinates space.
1622             int x = view.getX();
1623             int y = view.getY();
1624             g.translate(x, y);
1625             g.setClip(clip.x - x, clip.y - y, clip.width, clip.height);
1626             view.paintForceDoubleBuffered(g);
1627             g.translate(-x, -y);
1628             g.setClip(clip.x, clip.y, clip.width, clip.height);
1629         }
1630         else {
1631             // To avoid any problems that may result from the viewport being
1632             // bigger than the view we start painting from the viewport.
1633             try {
1634                 inBlitPaint = true;
1635                 paintForceDoubleBuffered(g);
1636             } finally {
1637                 inBlitPaint = false;
1638             }
1639         }
1640     }

1642     /**
1643      * Returns true if the viewport is not obscured by one of its ancestors,
1644      * or its ancestors children and if the viewport is showing. Blitting
1645      * when the view isn't showing will work,
1646      * or rather copyArea will work,
1647      * but will not produce the expected behavior.
1648      */
1649     private boolean canUseWindowBlitter() {
1650         if (!isShowing() || (!(getParent() instanceof JComponent) &&
1651                              !(getView() instanceof JComponent))) {
1652             return false;
1653         }
1654         if (isPainting()) {
1655             // We're in the process of painting, don't blit. If we were
1656             // to blit we would draw on top of what we're already drawing,
1657             // so bail.
1658             return false;
1659         }
1660 
1661         Rectangle dirtyRegion = RepaintManager.currentManager(this).
1662                                 getDirtyRegion((JComponent)getParent());
1663 
1664         if (dirtyRegion != null && dirtyRegion.width > 0 &&
1665             dirtyRegion.height > 0) {
1666             // Part of the scrollpane needs to be repainted too, don't blit.
1667             return false;
1668         }
1669 
1670         Rectangle clip = new Rectangle(0,0,getWidth(),getHeight());
1671         Rectangle oldClip = new Rectangle();
1672         Rectangle tmp2 = null;
1673         Container parent;
1674         Component lastParent = null;
1675         int x, y, w, h;
1676 
1677         for(parent = this; parent != null && isLightweightComponent(parent); parent = parent.getParent()) {
1678             x = parent.getX();
1679             y = parent.getY();
1680             w = parent.getWidth();
1681             h = parent.getHeight();
1682 
1683             oldClip.setBounds(clip);
1684             SwingUtilities.computeIntersection(0, 0, w, h, clip);
1685             if(!clip.equals(oldClip))
1686                 return false;
1687 
1688             if(lastParent != null && parent instanceof JComponent &&
1689                !((JComponent)parent).isOptimizedDrawingEnabled()) {
1690                 Component comps[] = parent.getComponents();
1691                 int index = 0;
1692 
1693                 for(int i = comps.length - 1 ;i >= 0; i--) {
1694                     if(comps[i] == lastParent) {
1695                         index = i - 1;
1696                         break;
1697                     }
1698                 }
1699 
1700                 while(index >= 0) {
1701                     tmp2 = comps[index].getBounds(tmp2);
1702 
1703                     if(tmp2.intersects(clip))
1704                         return false;
1705                     index--;
1706                 }
1707             }
1708             clip.x += x;
1709             clip.y += y;
1710             lastParent = parent;
1711         }
1712         if (parent == null) {
1713             // No Window parent.
1714             return false;
1715         }
1716         return true;
1717     }
