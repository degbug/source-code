  40 /**
  41  * An implementation of {@link ReadWriteLock} supporting similar
  42  * semantics to {@link ReentrantLock}.
  43  * <p>This class has the following properties:
  44  *
  45  * <ul>
  46  * <li><b>Acquisition order</b>
  47  *
  48  * <p>This class does not impose a reader or writer preference
  49  * ordering for lock access.  However, it does support an optional
  50  * <em>fairness</em> policy.
  51  *
  52  * <dl>
  53  * <dt><b><i>Non-fair mode (default)</i></b>
  54  * <dd>When constructed as non-fair (the default), the order of entry
  55  * to the read and write lock is unspecified, subject to reentrancy
  56  * constraints.  A nonfair lock that is continuously contended may
  57  * indefinitely postpone one or more reader or writer threads, but
  58  * will normally have higher throughput than a fair lock.
  59  *
  60  * <dt><b><i>Fair mode</i></b>
  61  * <dd>When constructed as fair, threads contend for entry using an
  62  * approximately arrival-order policy. When the currently held lock
  63  * is released, either the longest-waiting single writer thread will
  64  * be assigned the write lock, or if there is a group of reader threads
  65  * waiting longer than all waiting writer threads, that group will be
  66  * assigned the read lock.
  67  *
  68  * <p>A thread that tries to acquire a fair read lock (non-reentrantly)
  69  * will block if either the write lock is held, or there is a waiting
  70  * writer thread. The thread will not acquire the read lock until
  71  * after the oldest currently waiting writer thread has acquired and
  72  * released the write lock. Of course, if a waiting writer abandons
  73  * its wait, leaving one or more reader threads as the longest waiters
  74  * in the queue with the write lock free, then those readers will be
  75  * assigned the read lock.
  76  *
  77  * <p>A thread that tries to acquire a fair write lock (non-reentrantly)
  78  * will block unless both the read lock and write lock are free (which
  79  * implies there are no waiting threads).  (Note that the non-blocking
  80  * {@link ReadLock#tryLock()} and {@link WriteLock#tryLock()} methods
  81  * do not honor this fair setting and will immediately acquire the lock
  82  * if it is possible, regardless of waiting threads.)
  83  * <p>
  84  * </dl>
  85  *
  86  * <li><b>Reentrancy</b>
  87  *
  88  * <p>This lock allows both readers and writers to reacquire read or
  89  * write locks in the style of a {@link ReentrantLock}. Non-reentrant
  90  * readers are not allowed until all write locks held by the writing
  91  * thread have been released.
  92  *
  93  * <p>Additionally, a writer can acquire the read lock, but not
  94  * vice-versa.  Among other applications, reentrancy can be useful
  95  * when write locks are held during calls or callbacks to methods that
  96  * perform reads under read locks.  If a reader tries to acquire the
  97  * write lock it will never succeed.
  98  *
  99  * <li><b>Lock downgrading</b>
 100  * <p>Reentrancy also allows downgrading from the write lock to a read lock,
 101  * by acquiring the write lock, then the read lock and then releasing the
 102  * write lock. However, upgrading from a read lock to the write lock is
 103  * <b>not</b> possible.
 104  *
 105  * <li><b>Interruption of lock acquisition</b>
 106  * <p>The read lock and write lock both support interruption during lock
 107  * acquisition.
 108  *
 109  * <li><b>{@link Condition} support</b>
 110  * <p>The write lock provides a {@link Condition} implementation that
 111  * behaves in the same way, with respect to the write lock, as the
 112  * {@link Condition} implementation provided by
 113  * {@link ReentrantLock#newCondition} does for {@link ReentrantLock}.
 114  * This {@link Condition} can, of course, only be used with the write lock.
 115  *
 116  * <p>The read lock does not support a {@link Condition} and
 117  * {@code readLock().newCondition()} throws
 118  * {@code UnsupportedOperationException}.
 119  *
 120  * <li><b>Instrumentation</b>
 121  * <p>This class supports methods to determine whether locks
 122  * are held or contended. These methods are designed for monitoring
 123  * system state, not for synchronization control.
 124  * </ul>
 125  *
 126  * <p>Serialization of this class behaves in the same way as built-in
 127  * locks: a deserialized lock is in the unlocked state, regardless of
 128  * its state when serialized.
 129  *
 130  * <p><b>Sample usages</b>. Here is a code sketch showing how to perform
 131  * lock downgrading after updating a cache (exception handling is
 132  * particularly tricky when handling multiple locks in a non-nested
 133  * fashion):
 134  *
 135  * <pre> {@code
 136  * class CachedData {
 137  *   Object data;
 138  *   volatile boolean cacheValid;
 139  *   final ReentrantReadWriteLock rwl = new ReentrantReadWriteLock();
 140  *
 141  *   void processCachedData() {
 142  *     rwl.readLock().lock();
 143  *     if (!cacheValid) {
 144  *       // Must release read lock before acquiring write lock
 145  *       rwl.readLock().unlock();
 146  *       rwl.writeLock().lock();
 147  *       try {
 148  *         // Recheck state because another thread might have
 149  *         // acquired write lock and changed state before we did.
 150  *         if (!cacheValid) {
 151  *           data = ...
 152  *           cacheValid = true;
 153  *         }
 154  *         // Downgrade by acquiring read lock before releasing write lock
 155  *         rwl.readLock().lock();
 156  *       } finally {
 157  *         rwl.writeLock().unlock(); // Unlock write, still hold read
 158  *       }
 159  *     }
 160  *
 161  *     try {
 162  *       use(data);
 163  *     } finally {
 164  *       rwl.readLock().unlock();
 165  *     }
 166  *   }
 167  * }}</pre>
 168  *
 169  * ReentrantReadWriteLocks can be used to improve concurrency in some
 170  * uses of some kinds of Collections. This is typically worthwhile
 171  * only when the collections are expected to be large, accessed by
 172  * more reader threads than writer threads, and entail operations with
 173  * overhead that outweighs synchronization overhead. For example, here
 174  * is a class using a TreeMap that is expected to be large and
 175  * concurrently accessed.
 176  *
 177  *  <pre> {@code
 178  * class RWDictionary {
 179  *   private final Map<String, Data> m = new TreeMap<String, Data>();
 180  *   private final ReentrantReadWriteLock rwl = new ReentrantReadWriteLock();
 181  *   private final Lock r = rwl.readLock();
 182  *   private final Lock w = rwl.writeLock();
 183  *
 184  *   public Data get(String key) {
 185  *     r.lock();
 186  *     try { return m.get(key); }
 187  *     finally { r.unlock(); }
 188  *   }
 189  *   public String[] allKeys() {
 190  *     r.lock();
 191  *     try { return m.keySet().toArray(); }
 192  *     finally { r.unlock(); }
 193  *   }
 194  *   public Data put(String key, Data value) {
 195  *     w.lock();
 196  *     try { return m.put(key, value); }
 197  *     finally { w.unlock(); }
 198  *   }
 199  *   public void clear() {
 200  *     w.lock();
 201  *     try { m.clear(); }
 202  *     finally { w.unlock(); }
 203  *   }
 204  * }}</pre>
 205  *

 209  * and 65535 read locks. Attempts to exceed these limits result in
 210  * {@link Error} throws from locking methods.
 211  *

 213  * @author Doug Lea
 214  */

 255         /*
 256          * Read vs write count extraction constants and functions.
 257          * Lock state is logically divided into two unsigned shorts:
 258          * The lower one representing the exclusive (writer) lock hold count,
 259          * and the upper the shared (reader) hold count.
 260          */
 261 
 262         static final int SHARED_SHIFT   = 16;
 263         static final int SHARED_UNIT    = (1 << SHARED_SHIFT);
 264         static final int MAX_COUNT      = (1 << SHARED_SHIFT) - 1;
 265         static final int EXCLUSIVE_MASK = (1 << SHARED_SHIFT) - 1;
 266 
 267         /** Returns the number of shared holds represented in count  */
 268         static int sharedCount(int c)    { return c >>> SHARED_SHIFT; }
 269         /** Returns the number of exclusive holds represented in count  */
 270         static int exclusiveCount(int c) { return c & EXCLUSIVE_MASK; }

 369         protected final boolean tryRelease(int releases) {
 370             if (!isHeldExclusively())
 371                 throw new IllegalMonitorStateException();
 372             int nextc = getState() - releases;
 373             boolean free = exclusiveCount(nextc) == 0;
 374             if (free)
 375                 setExclusiveOwnerThread(null);
 376             setState(nextc);
 377             return free;
 378         }

 380         protected final boolean tryAcquire(int acquires) {
 381             /*
 382              * Walkthrough:
 383              * 1. If read count nonzero or write count nonzero
 384              *    and owner is a different thread, fail.
 385              * 2. If count would saturate, fail. (This can only
 386              *    happen if count is already nonzero.)
 387              * 3. Otherwise, this thread is eligible for lock if
 388              *    it is either a reentrant acquire or
 389              *    queue policy allows it. If so, update state
 390              *    and set owner.
 391              */
 392             Thread current = Thread.currentThread();
 393             int c = getState();
 394             int w = exclusiveCount(c);
 395             if (c != 0) {
 396                 // (Note: if c != 0 and w == 0 then shared count != 0)
 397                 if (w == 0 || current != getExclusiveOwnerThread())
 398                     return false;
 399                 if (w + exclusiveCount(acquires) > MAX_COUNT)
 400                     throw new Error("Maximum lock count exceeded");
 401                 // Reentrant acquire
 402                 setState(c + acquires);
 403                 return true;
 404             }
 405             if (writerShouldBlock() ||
 406                 !compareAndSetState(c, c + acquires))
 407                 return false;
 408             setExclusiveOwnerThread(current);
 409             return true;
 410         }

 412         protected final boolean tryReleaseShared(int unused) {
 413             Thread current = Thread.currentThread();
 414             if (firstReader == current) {
 415                 // assert firstReaderHoldCount > 0;
 416                 if (firstReaderHoldCount == 1)
 417                     firstReader = null;
 418                 else
 419                     firstReaderHoldCount--;
 420             } else {
 421                 HoldCounter rh = cachedHoldCounter;
 422                 if (rh == null || rh.tid != getThreadId(current))
 423                     rh = readHolds.get();
 424                 int count = rh.count;
 425                 if (count <= 1) {
 426                     readHolds.remove();
 427                     if (count <= 0)
 428                         throw unmatchedUnlockException();
 429                 }
 430                 --rh.count;
 431             }
 432             for (;;) {
 433                 int c = getState();
 434                 int nextc = c - SHARED_UNIT;
 435                 if (compareAndSetState(c, nextc))
 436                     // Releasing the read lock has no effect on readers,
 437                     // but it may allow waiting writers to proceed if
 438                     // both read and write locks are now free.
 439                     return nextc == 0;
 440             }
 441         }

 443         private IllegalMonitorStateException unmatchedUnlockException() {
 444             return new IllegalMonitorStateException(
 445                 "attempt to unlock read lock, not locked by current thread");
 446         }

 448         protected final int tryAcquireShared(int unused) {
 449             /*
 450              * Walkthrough:
 451              * 1. If write lock held by another thread, fail.
 452              * 2. Otherwise, this thread is eligible for
 453              *    lock wrt state, so ask if it should block
 454              *    because of queue policy. If not, try
 455              *    to grant by CASing state and updating count.
 456              *    Note that step does not check for reentrant
 457              *    acquires, which is postponed to full version
 458              *    to avoid having to check hold count in
 459              *    the more typical non-reentrant case.
 460              * 3. If step 2 fails either because thread
 461              *    apparently not eligible or CAS fails or count
 462              *    saturated, chain to version with full retry loop.
 463              */
 464             Thread current = Thread.currentThread();
 465             int c = getState();
 466             if (exclusiveCount(c) != 0 &&
 467                 getExclusiveOwnerThread() != current)
 468                 return -1;
 469             int r = sharedCount(c);
 470             if (!readerShouldBlock() &&
 471                 r < MAX_COUNT &&
 472                 compareAndSetState(c, c + SHARED_UNIT)) {
 473                 if (r == 0) {
 474                     firstReader = current;
 475                     firstReaderHoldCount = 1;
 476                 } else if (firstReader == current) {
 477                     firstReaderHoldCount++;
 478                 } else {
 479                     HoldCounter rh = cachedHoldCounter;
 480                     if (rh == null || rh.tid != getThreadId(current))
 481                         cachedHoldCounter = rh = readHolds.get();
 482                     else if (rh.count == 0)
 483                         readHolds.set(rh);
 484                     rh.count++;
 485                 }
 486                 return 1;
 487             }
 488             return fullTryAcquireShared(current);
 489         }

 491         /**
 492          * Full version of acquire for reads, that handles CAS misses
 493          * and reentrant reads not dealt with in tryAcquireShared.
 494          */
 495         final int fullTryAcquireShared(Thread current) {
 496             /*
 497              * This code is in part redundant with that in
 498              * tryAcquireShared but is simpler overall by not
 499              * complicating tryAcquireShared with interactions between
 500              * retries and lazily reading hold counts.
 501              */
 502             HoldCounter rh = null;
 503             for (;;) {
 504                 int c = getState();
 505                 if (exclusiveCount(c) != 0) {
 506                     if (getExclusiveOwnerThread() != current)
 507                         return -1;
 508                     // else we hold the exclusive lock; blocking here
 509                     // would cause deadlock.
 510                 } else if (readerShouldBlock()) {
 511                     // Make sure we're not acquiring read lock reentrantly
 512                     if (firstReader == current) {
 513                         // assert firstReaderHoldCount > 0;
 514                     } else {
 515                         if (rh == null) {
 516                             rh = cachedHoldCounter;
 517                             if (rh == null || rh.tid != getThreadId(current)) {
 518                                 rh = readHolds.get();
 519                                 if (rh.count == 0)
 520                                     readHolds.remove();
 521                             }
 522                         }
 523                         if (rh.count == 0)
 524                             return -1;
 525                     }
 526                 }
 527                 if (sharedCount(c) == MAX_COUNT)
 528                     throw new Error("Maximum lock count exceeded");
 529                 if (compareAndSetState(c, c + SHARED_UNIT)) {
 530                     if (sharedCount(c) == 0) {
 531                         firstReader = current;
 532                         firstReaderHoldCount = 1;
 533                     } else if (firstReader == current) {
 534                         firstReaderHoldCount++;
 535                     } else {
 536                         if (rh == null)
 537                             rh = cachedHoldCounter;
 538                         if (rh == null || rh.tid != getThreadId(current))
 539                             rh = readHolds.get();
 540                         else if (rh.count == 0)
 541                             readHolds.set(rh);
 542                         rh.count++;
 543                         cachedHoldCounter = rh; // cache for release
 544                     }
 545                     return 1;
 546                 }
 547             }
 548         }

 550         /**
 551          * Performs tryLock for write, enabling barging in both modes.
 552          * This is identical in effect to tryAcquire except for lack
 553          * of calls to writerShouldBlock.
 554          */
 555         final boolean tryWriteLock() {
 556             Thread current = Thread.currentThread();
 557             int c = getState();
 558             if (c != 0) {
 559                 int w = exclusiveCount(c);
 560                 if (w == 0 || current != getExclusiveOwnerThread())
 561                     return false;
 562                 if (w == MAX_COUNT)
 563                     throw new Error("Maximum lock count exceeded");
 564             }
 565             if (!compareAndSetState(c, c + 1))
 566                 return false;
 567             setExclusiveOwnerThread(current);
 568             return true;
 569         }

 571         /**
 572          * Performs tryLock for read, enabling barging in both modes.
 573          * This is identical in effect to tryAcquireShared except for
 574          * lack of calls to readerShouldBlock.
 575          */
 576         final boolean tryReadLock() {
 577             Thread current = Thread.currentThread();
 578             for (;;) {
 579                 int c = getState();
 580                 if (exclusiveCount(c) != 0 &&
 581                     getExclusiveOwnerThread() != current)
 582                     return false;
 583                 int r = sharedCount(c);
 584                 if (r == MAX_COUNT)
 585                     throw new Error("Maximum lock count exceeded");
 586                 if (compareAndSetState(c, c + SHARED_UNIT)) {
 587                     if (r == 0) {
 588                         firstReader = current;
 589                         firstReaderHoldCount = 1;
 590                     } else if (firstReader == current) {
 591                         firstReaderHoldCount++;
 592                     } else {
 593                         HoldCounter rh = cachedHoldCounter;
 594                         if (rh == null || rh.tid != getThreadId(current))
 595                             cachedHoldCounter = rh = readHolds.get();
 596                         else if (rh.count == 0)
 597                             readHolds.set(rh);
 598                         rh.count++;
 599                     }
 600                     return true;
 601                 }
 602             }

 636         final int getReadHoldCount() {
 637             if (getReadLockCount() == 0)
 638                 return 0;
 639 
 640             Thread current = Thread.currentThread();
 641             if (firstReader == current)
 642                 return firstReaderHoldCount;
 643 
 644             HoldCounter rh = cachedHoldCounter;
 645             if (rh != null && rh.tid == getThreadId(current))
 646                 return rh.count;
 647 
 648             int count = readHolds.get().count;
 649             if (count == 0) readHolds.remove();
