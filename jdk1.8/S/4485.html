  30 /**
  31  * The facility that a disconnected RowSet object calls on
  32  * to populate itself with rows of data. A reader (an object implementing the
  33  * RowSetReader interface) may be registered with
  34  * a RowSet object that supports the reader/writer paradigm.
  35  * When the RowSet object's execute method is
  36  * called, it in turn calls the reader's readData method.
  37  *
  38  * @since 1.4
  39  */
  40 
  41 public interface RowSetReader {
  42 
  43   /**
  44    * Reads the new contents of the calling RowSet object.
  45    * In order to call this method, a RowSet
  46    * object must have implemented the RowSetInternal interface
  47    * and registered this RowSetReader object as its reader.
  48    * The readData  method is invoked internally
  49    * by the RowSet.execute method for rowsets that support the
  50    * reader/writer paradigm.
  51    *
  52    * 
The readData method adds rows to the caller.

  53    * It can be implemented in a wide variety of ways and can even
  54    * populate the caller with rows from a nonrelational data source.
  55    * In general, a reader may invoke any of the rowset's methods,
  56    * with one exception. Calling the method execute will
  57    * cause an SQLException to be thrown
  58    * because execute may not be called recursively.  Also,
  59    * when a reader invokes RowSet methods, no listeners
  60    * are notified; that is, no RowSetEvent objects are
  61    * generated and no RowSetListener methods are invoked.
  62    * This is true because listeners are already being notified by the method
  63    * execute.
  64    *
  65    * @param caller the RowSet object (1) that has implemented the
  66    *         RowSetInternal interface, (2) with which this reader is
  67    *        registered, and (3) whose execute method called this reader
  68    * @exception SQLException if a database access error occurs or this method
  69    *            invokes the RowSet.execute method
  70    */
  71   void readData(RowSetInternal caller) throws SQLException;
  72 
  73 }
