  48 /**
  49  * The TextAttribute class defines attribute keys and
  50  * attribute values used for text rendering.
  51  * 
  52  * TextAttribute instances are used as attribute keys to
  53  * identify attributes in
  54  * {@link java.awt.Font Font},
  55  * {@link java.awt.font.TextLayout TextLayout},
  56  * {@link java.text.AttributedCharacterIterator AttributedCharacterIterator},
  57  * and other classes handling text attributes. Other constants defined
  58  * in this class can be used as attribute values.
  59  * 
  60  * For each text attribute, the documentation provides:
  61  * 
    
  62  *   
  • the type of its value,
    • 
  63  *   
  • the relevant predefined constants, if any
    • 
  64  *   
  • the default effect if the attribute is absent
    • 
  65  *   
  • the valid values if there are limitations
    • 
  66  *   
  • a description of the effect.
    • 
  67  * 
  68  * 
  69  * 
    Values
    
  70  * 
      
  71  *   
    • The values of attributes must always be immutable.
      • 
  72  *   
    • Where value limitations are given, any value outside of that
      • 
  73  *   set is reserved for future use; the value will be treated as
  74  *   the default.
  75  *   
    • The value null is treated the same as the
      • 
  76  *   default value and results in the default behavior.
  77  *   
    • If the value is not of the proper type, the attribute
      • 
  78  *   will be ignored.
  79  *   
    • The identity of the value does not matter, only the actual
      • 
  80  *   value.  For example, TextAttribute.WEIGHT_BOLD and
  81  *   new Float(2.0)
  82  *   indicate the same WEIGHT.
  83  *   
    • Attribute values of type Number (used for
      • 
  84  *   WEIGHT, WIDTH, POSTURE,
  85  *   SIZE, JUSTIFICATION, and
  86  *   TRACKING) can vary along their natural range and are
  87  *   not restricted to the predefined constants.
  88  *   Number.floatValue() is used to get the actual value
  89  *   from the Number.
  90  *   
    • The values for WEIGHT, WIDTH, and
      • 
  91  *   POSTURE are interpolated by the system, which
  92  *   can select the 'nearest available' font or use other techniques to
  93  *   approximate the user's request.

 276     protected Object readResolve() throws InvalidObjectException {
 277         if (this.getClass() != TextAttribute.class) {
 278             throw new InvalidObjectException(
 279                 "subclass didn't correctly implement readResolve");
 280         }
 281 
 282         TextAttribute instance = instanceMap.get(getName());
 283         if (instance != null) {
 284             return instance;
 285         } else {
 286             throw new InvalidObjectException("unknown attribute name");
 287         }
 288     }

 543     /**
 544      * Attribute key for the transform of a font.  Values are
 545      * instances of TransformAttribute.  The
 546      * default value is TransformAttribute.IDENTITY.
 547      *
 548      * The TransformAttribute class defines the
 549      * constant {@link TransformAttribute#IDENTITY IDENTITY}.
 550      *
 551      * This corresponds to the transform passed to
 552      * Font.deriveFont(AffineTransform).  Since that
 553      * transform is mutable and TextAttribute values must
 554      * not be, the TransformAttribute wrapper class is
 555      * used.
 556      *
 557      * The primary intent is to support scaling and skewing, though
 558      * other effects are possible.
 559      *
 560      * Some transforms will cause the baseline to be rotated and/or
 561      * shifted.  The text and the baseline are transformed together so
 562      * that the text follows the new baseline.  For example, with text
 563      * on a horizontal baseline, the new baseline follows the
 564      * direction of the unit x vector passed through the
 565      * transform. Text metrics are measured against this new baseline.
 566      * So, for example, with other things being equal, text rendered
 567      * with a rotated TRANSFORM and an unrotated TRANSFORM will measure as
 568      * having the same ascent, descent, and advance.
 569      *
 570      * In styled text, the baselines for each such run are aligned
 571      * one after the other to potentially create a non-linear baseline
 572      * for the entire run of text. For more information, see {@link
 573      * TextLayout#getLayoutPath}.
 574      *
 575      * @see TransformAttribute
 576      * @see java.awt.geom.AffineTransform
 577      */

 579         new TextAttribute("transform");

 616     /**
 617      * Attribute key used to provide the font to use to render text.
 618      * Values are instances of {@link java.awt.Font}.  The default
 619      * value is null, indicating that normal resolution of a
 620      * Font from attributes should be performed.
 621      *
 622      * TextLayout and
 623      * AttributedCharacterIterator work in terms of
 624      * Maps of TextAttributes.  Normally,
 625      * all the attributes are examined and used to select and
 626      * configure a Font instance.  If a FONT
 627      * attribute is present, though, its associated Font
 628      * will be used.  This provides a way for users to override the
 629      * resolution of font attributes into a Font, or
 630      * force use of a particular Font instance.  This
 631      * also allows users to specify subclasses of Font in
 632      * cases where a Font can be subclassed.
 633      *
 634      * FONT is used for special situations where
 635      * clients already have a Font instance but still
 636      * need to use Map-based APIs.  Typically, there will
 637      * be no other attributes in the Map except the
 638      * FONT attribute.  With Map-based APIs
 639      * the common case is to specify all attributes individually, so
 640      * FONT is not needed or desireable.
 641      *
 642      * However, if both FONT and other attributes are
 643      * present in the Map, the rendering system will
 644      * merge the attributes defined in the Font with the
 645      * additional attributes.  This merging process classifies
 646      * TextAttributes into two groups.  One group, the
 647      * 'primary' group, is considered fundamental to the selection and
 648      * metric behavior of a font.  These attributes are
 649      * FAMILY, WEIGHT, WIDTH,
 650      * POSTURE, SIZE,
 651      * TRANSFORM, SUPERSCRIPT, and
 652      * TRACKING. The other group, the 'secondary' group,
 653      * consists of all other defined attributes, with the exception of
 654      * FONT itself.
 655      *
 656      * To generate the new Map, first the
 657      * Font is obtained from the FONT
 658      * attribute, and all of its attributes extracted into a
 659      * new Map.  Then only the secondary
 660      * attributes from the original Map are added to
 661      * those in the new Map.  Thus the values of primary
 662      * attributes come solely from the Font, and the
 663      * values of secondary attributes originate with the
 664      * Font but can be overridden by other values in the
 665      * Map.
 666      *
 667      * Note:Font's Map-based
 668      * constructor and deriveFont methods do not process
 669      * the FONT attribute, as these are used to create
 670      * new Font objects.  Instead, {@link
 671      * java.awt.Font#getFont(Map) Font.getFont(Map)} should be used to
 672      * handle the FONT attribute.
 673      *
 674      * @see java.awt.Font
 675      */
 676     public static final TextAttribute FONT =
 677         new TextAttribute("font");
