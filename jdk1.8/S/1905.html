  26 /**
  27  * Interfaces and classes providing access to file and file system attributes.
  28  *
  29  * 

  30  * Attribute viewsDescription
  31  * {@link java.nio.file.attribute.AttributeView}
  32  *     Can read or update non-opaque values associated with objects in a file system
  33  *   {@link java.nio.file.attribute.FileAttributeView}
  34  *     Can read or update file attributes
  35  *     {@link java.nio.file.attribute.BasicFileAttributeView}  
  36  *     Can read or update a basic set of file attributes
  37  *       {@link java.nio.file.attribute.PosixFileAttributeView}  
  38  *     Can read or update POSIX defined file attributes
  39  *       {@link java.nio.file.attribute.DosFileAttributeView}  
  40  *     Can read or update FAT file attributes
  41  *     {@link java.nio.file.attribute.FileOwnerAttributeView}  
  42  *     Can read or update the owner of a file
  43  *      {@link java.nio.file.attribute.AclFileAttributeView}  
  44  *     Can read or update Access Control Lists
  45  *     {@link java.nio.file.attribute.UserDefinedFileAttributeView}  
  46  *     Can read or update user-defined file attributes
  47  *   {@link java.nio.file.attribute.FileStoreAttributeView}
  48  *     Can read or update file system attributes
  49  * 
  50  *
  51  *  An attribute view provides a read-only or updatable view of the non-opaque
  52  * values, or metadata, associated with objects in a file system.
  53  * The {@link java.nio.file.attribute.FileAttributeView} interface is
  54  * extended by several other interfaces that that views to specific sets of file
  55  * attributes. {@code FileAttributeViews} are selected by invoking the {@link
  56  * java.nio.file.Files#getFileAttributeView} method with a
  57  * type-token to identify the required view. Views can also be identified
  58  * by name. The {@link java.nio.file.attribute.FileStoreAttributeView} interface
  59  * provides access to file store attributes. A {@code FileStoreAttributeView} of
  60  * a given type is obtained by invoking the {@link
  61  * java.nio.file.FileStore#getFileStoreAttributeView} method.
  62  *
  63  *  The {@link java.nio.file.attribute.BasicFileAttributeView}
  64  * class defines methods to read and update a basic set of file
  65  * attributes that are common to many file systems.
  66  *
  67  *  The {@link java.nio.file.attribute.PosixFileAttributeView}
  68  * interface extends {@code BasicFileAttributeView} by defining methods
  69  * to access the file attributes commonly used by file systems and operating systems
  70  * that implement the Portable Operating System Interface (POSIX) family of
  71  * standards.
  72  *
  73  *  The {@link java.nio.file.attribute.DosFileAttributeView}
  74  * class extends {@code BasicFileAttributeView} by defining methods to
  75  * access the legacy "DOS" file attributes supported on file systems such as File
  76  * Allocation Tabl (FAT), commonly used in consumer devices.
  77  *
  78  *  The {@link java.nio.file.attribute.AclFileAttributeView}
  79  * class defines methods to read and write the Access Control List (ACL)
  80  * file attribute. The ACL model used by this file attribute view is based
  81  * on the model defined by 
  82  * RFC 3530: Network File System (NFS) version 4 Protocol.
  83  *
  84  *  In addition to attribute views, this package also defines classes and
  85  * interfaces that are used when accessing attributes:
  86  *
  87  * 
    
  88  *
  89  *   
  •  The {@link java.nio.file.attribute.UserPrincipal} and
    • 
  90  *   {@link java.nio.file.attribute.GroupPrincipal} interfaces represent an
  91  *   identity or group identity. 
  92  *
  93  *   
  •  The {@link java.nio.file.attribute.UserPrincipalLookupService}
    • 
  94  *   interface defines methods to lookup user or group principals. 
  95  *
  96  *   
  •  The {@link java.nio.file.attribute.FileAttribute} interface
    • 
  97  *   represents the value of an attribute for cases where the attribute value is
  98  *   required to be set atomically when creating an object in the file system. 
  99  *
 100  * 
 101  *
 102  *
 103  *  Unless otherwise noted, passing a null argument to a constructor
 104  * or method in any class or interface in this package will cause a {@link
 105  * java.lang.NullPointerException NullPointerException} to be thrown.
 106  *
 107  * @since 1.7
 108  */
