  38 /**
  39  * This interface specifies the behavior to be implemented by an
  40  * MBean Server Interceptor.  An MBean Server Interceptor has
  41  * essentially the same interface as an MBean Server.  An MBean Server
  42  * forwards received requests to its default interceptor, which may
  43  * handle them itself or forward them to other interceptors.  The
  44  * default interceptor may be changed via the {@link
  45  * com.sun.jmx.mbeanserver.SunJmxMBeanServer#setMBeanServerInterceptor}
  46  * method.
  47  *
  48  * The initial default interceptor provides the standard MBean
  49  * Server behavior.  It handles a collection of named MBeans, each
  50  * represented by a Java object.  A replacement default interceptor
  51  * may build on this behavior, for instance by adding logging or
  52  * security checks, before forwarding requests to the initial default
  53  * interceptor.  Or, it may route each request to one of a number of
  54  * sub-interceptors, for instance based on the {@link ObjectName} in
  55  * the request.
  56  *
  57  * An interceptor, default or not, need not implement MBeans as
  58  * Java objects, in the way that the initial default interceptor does.
  59  * It may instead implement virtual MBeans, which do not
  60  * exist as Java objects when they are not in use.  For example, these
  61  * MBeans could be implemented by forwarding requests to a database,
  62  * or to a remote MBean server, or by performing system calls to query
  63  * or modify system resources.
  64  *
  65  * @since 1.5
  66  */
