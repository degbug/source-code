  63 /**
  64  * A class loader is an object that is responsible for loading classes. The
  65  * class ClassLoader is an abstract class.  Given the 
  66  * href="#name">binary name of a class, a class loader should attempt to
  67  * locate or generate data that constitutes a definition for the class.  A
  68  * typical strategy is to transform the name into a file name and then read a
  69  * "class file" of that name from a file system.
  70  *
  71  *  Every {@link Class Class} object contains a {@link
  72  * Class#getClassLoader() reference} to the ClassLoader that defined
  73  * it.
  74  *
  75  *  Class objects for array classes are not created by class
  76  * loaders, but are created automatically as required by the Java runtime.
  77  * The class loader for an array class, as returned by {@link
  78  * Class#getClassLoader()} is the same as the class loader for its element
  79  * type; if the element type is a primitive type, then the array class has no
  80  * class loader.
  81  *
  82  *  Applications implement subclasses of ClassLoader in order to
  83  * extend the manner in which the Java virtual machine dynamically loads
  84  * classes.
  85  *
  86  *  Class loaders may typically be used by security managers to indicate
  87  * security domains.
  88  *
  89  *  The ClassLoader class uses a delegation model to search for
  90  * classes and resources.  Each instance of ClassLoader has an
  91  * associated parent class loader.  When requested to find a class or
  92  * resource, a ClassLoader instance will delegate the search for the
  93  * class or resource to its parent class loader before attempting to find the
  94  * class or resource itself.  The virtual machine's built-in class loader,
  95  * called the "bootstrap class loader", does not itself have a parent but may
  96  * serve as the parent of a ClassLoader instance.
  97  *
  98  *  Class loaders that support concurrent loading of classes are known as
  99  * parallel capable class loaders and are required to register
 100  * themselves at their class initialization time by invoking the
 101  * {@link
 102  * #registerAsParallelCapable ClassLoader.registerAsParallelCapable}
 103  * method. Note that the ClassLoader class is registered as parallel
 104  * capable by default. However, its subclasses still need to register themselves
 105  * if they are parallel capable. 

 106  * In environments in which the delegation model is not strictly
 107  * hierarchical, class loaders need to be parallel capable, otherwise class
 108  * loading can lead to deadlocks because the loader lock is held for the
 109  * duration of the class loading process (see {@link #loadClass
 110  * loadClass} methods).
 111  *
 112  *  Normally, the Java virtual machine loads classes from the local file
 113  * system in a platform-dependent manner.  For example, on UNIX systems, the
 114  * virtual machine loads classes from the directory defined by the
 115  * CLASSPATH environment variable.
 116  *
 117  *  However, some classes may not originate from a file; they may originate
 118  * from other sources, such as the network, or they could be constructed by an
 119  * application.  The method {@link #defineClass(String, byte[], int, int)
 120  * defineClass} converts an array of bytes into an instance of class
 121  * Class. Instances of this newly defined class can be created using
 122  * {@link Class#newInstance Class.newInstance}.
 123  *
 124  *  The methods and constructors of objects created by a class loader may
 125  * reference other classes.  To determine the class(es) referred to, the Java
 126  * virtual machine invokes the {@link #loadClass loadClass} method of
 127  * the class loader that originally created the class.
 128  *
 129  *  For example, an application could create a network class loader to
 130  * download class files from a server.  Sample code might look like:
 131  *
 132  * 

 133  *   ClassLoader loader = new NetworkClassLoader(host, port);
 134  *   Object main = loader.loadClass("Main", true).newInstance();
 135  *        . . .
 136  * 
 137  *
 138  *  The network class loader subclass must define the methods {@link
 139  * #findClass findClass} and loadClassData to load a class
 140  * from the network.  Once it has downloaded the bytes that make up the class,
 141  * it should use the method {@link #defineClass defineClass} to
 142  * create a class instance.  A sample implementation is:
 143  *
 144  * 

 145  *     class NetworkClassLoader extends ClassLoader {
 146  *         String host;
 147  *         int port;
 148  *
 149  *         public Class findClass(String name) {
 150  *             byte[] b = loadClassData(name);
 151  *             return defineClass(name, b, 0, b.length);

 361      * Loads the class with the specified binary name.  The
 362      * default implementation of this method searches for classes in the
 363      * following order:
 364      *
 365      * 
    
 366      *
 367      *   
  1.  Invoke {@link #findLoadedClass(String)} to check if the class
    2. 
 368      *   has already been loaded.  
 369      *
 370      *   
  2.  Invoke the {@link #loadClass(String) loadClass} method
    3. 
 371      *   on the parent class loader.  If the parent is null the class
 372      *   loader built-in to the virtual machine is used, instead.  
 373      *
 374      *   
  3.  Invoke the {@link #findClass(String)} method to find the
    4. 
 375      *   class.  
 376      *
 377      * 
 378      *
 379      *  If the class was found using the above steps, and the
 380      * resolve flag is true, this method will then invoke the {@link
 381      * #resolveClass(Class)} method on the resulting Class object.
 382      *
 383      *  Subclasses of ClassLoader are encouraged to override {@link
 384      * #findClass(String)}, rather than this method.  
 385      *
 386      *  Unless overridden, this method synchronizes on the result of
 387      * {@link #getClassLoadingLock getClassLoadingLock} method
 388      * during the entire class loading process.
 389      *
 390      * @param  name
 391      *         The binary name of the class
 392      *
 393      * @param  resolve
 394      *         If true then resolve the class
 395      *
 396      * @return  The resulting Class object
 397      *
 398      * @throws  ClassNotFoundException
 399      *          If the class could not be found
 400      */
 401     protected Class<?> loadClass(String name, boolean resolve)
 402         throws ClassNotFoundException
 403     {
 404         synchronized (getClassLoadingLock(name)) {
 405             // First, check if the class has already been loaded
 406             Class<?> c = findLoadedClass(name);
 407             if (c == null) {
 408                 long t0 = System.nanoTime();
 409                 try {
 410                     if (parent != null) {
 411                         c = parent.loadClass(name, false);
 412                     } else {
 413                         c = findBootstrapClassOrNull(name);
 414                     }
 415                 } catch (ClassNotFoundException e) {
 416                     // ClassNotFoundException thrown if class not found
 417                     // from the non-null parent class loader
 418                 }
 419 
 420                 if (c == null) {
 421                     // If still not found, then invoke findClass in order
 422                     // to find the class.
 423                     long t1 = System.nanoTime();
 424                     c = findClass(name);
 425 
 426                     // this is the defining class loader; record the stats
 427                     sun.misc.PerfCounter.getParentDelegationTime().addTime(t1 - t0);
 428                     sun.misc.PerfCounter.getFindClassTime().addElapsedTimeFrom(t1);
 429                     sun.misc.PerfCounter.getFindClasses().increment();
 430                 }
 431             }
 432             if (resolve) {
 433                 resolveClass(c);
 434             }
 435             return c;
 436         }
 437     }

 874     private void checkCerts(String name, CodeSource cs) {
 875         int i = name.lastIndexOf('.');
 876         String pname = (i == -1) ? "" : name.substring(0, i);
 877 
 878         Certificate[] certs = null;
 879         if (cs != null) {
 880             certs = cs.getCertificates();
 881         }
 882         Certificate[] pcerts = null;
 883         if (parallelLockMap == null) {
 884             synchronized (this) {
 885                 pcerts = package2certs.get(pname);
 886                 if (pcerts == null) {
 887                     package2certs.put(pname, (certs == null? nocerts:certs));
 888                 }
 889             }
 890         } else {
 891             pcerts = ((ConcurrentHashMap<String, Certificate[]>)package2certs).
 892                 putIfAbsent(pname, (certs == null? nocerts:certs));
 893         }
 894         if (pcerts != null && !compareCerts(pcerts, certs)) {
 895             throw new SecurityException("class \""+ name +
 896                  "\"'s signer information does not match signer information of other classes in the same package");
 897         }
 898     }

 900     /**
 901      * check to make sure the certs for the new class (certs) are the same as
 902      * the certs for the first class inserted in the package (pcerts)
 903      */
 904     private boolean compareCerts(Certificate[] pcerts,
 905                                  Certificate[] certs)
 906     {
 907         // certs can be null, indicating no certs.
 908         if ((certs == null) || (certs.length == 0)) {
 909             return pcerts.length == 0;
 910         }
 911 
 912         // the length must be the same at this point
 913         if (certs.length != pcerts.length)
 914             return false;
 915 
 916         // go through and make sure all the certs in one array
 917         // are in the other and vice-versa.
 918         boolean match;
 919         for (int i = 0; i < certs.length; i++) {
 920             match = false;
 921             for (int j = 0; j < pcerts.length; j++) {
 922                 if (certs[i].equals(pcerts[j])) {
 923                     match = true;
 924                     break;
 925                 }
 926             }
 927             if (!match) return false;
 928         }
 929 
 930         // now do the same for pcerts
 931         for (int i = 0; i < pcerts.length; i++) {
 932             match = false;
 933             for (int j = 0; j < certs.length; j++) {
 934                 if (pcerts[i].equals(certs[j])) {
 935                     match = true;
 936                     break;
 937                 }
 938             }
 939             if (!match) return false;
 940         }
 941 
 942         return true;
 943     }

1779     private static String[] initializePath(String propname) {
1780         String ldpath = System.getProperty(propname, "");
1781         String ps = File.pathSeparator;
1782         int ldlen = ldpath.length();
1783         int i, j, n;
1784         // Count the separators in the path
1785         i = ldpath.indexOf(ps);
1786         n = 0;
1787         while (i >= 0) {
1788             n++;
1789             i = ldpath.indexOf(ps, i + 1);
1790         }
1791 
1792         // allocate the array of paths - n :'s = n + 1 path elements
1793         String[] paths = new String[n + 1];
1794 
1795         // Fill the array with paths from the ldpath
1796         n = i = 0;
1797         j = ldpath.indexOf(ps);
1798         while (j >= 0) {
1799             if (j - i > 0) {
1800                 paths[n++] = ldpath.substring(i, j);
1801             } else if (j - i == 0) {
1802                 paths[n++] = ".";
1803             }
1804             i = j + 1;
1805             j = ldpath.indexOf(ps, i);
1806         }
1807         paths[n] = ldpath.substring(i, ldlen);
1808         return paths;
1809     }

1811     // Invoked in the java.lang.Runtime class to implement load and loadLibrary.
1812     static void loadLibrary(Class<?> fromClass, String name,
1813                             boolean isAbsolute) {
1814         ClassLoader loader =
1815             (fromClass == null) ? null : fromClass.getClassLoader();

1821             if (loadLibrary0(fromClass, new File(name))) {
1822                 return;
1823             }

1825         }
1826         if (loader != null) {
1827             String libfilename = loader.findLibrary(name);
1828             if (libfilename != null) {
1829                 File libfile = new File(libfilename);

1831                     throw new UnsatisfiedLinkError(
1832     "ClassLoader.findLibrary failed to return an absolute path: " + libfilename);
1833                 }
1834                 if (loadLibrary0(fromClass, libfile)) {
1835                     return;
1836                 }
1837                 throw new UnsatisfiedLinkError("Can't load " + libfilename);
1838             }
1839         }
1840         for (int i = 0 ; i < sys_paths.length ; i++) {
1841             File libfile = new File(sys_paths[i], System.mapLibraryName(name));
1842             if (loadLibrary0(fromClass, libfile)) {
1843                 return;
1844             }
1845             libfile = ClassLoaderHelper.mapAlternativeName(libfile);
1846             if (libfile != null && loadLibrary0(fromClass, libfile)) {
1847                 return;
1848             }
1849         }
1850         if (loader != null) {
1851             for (int i = 0 ; i < usr_paths.length ; i++) {
1852                 File libfile = new File(usr_paths[i],
1853                                         System.mapLibraryName(name));
1854                 if (loadLibrary0(fromClass, libfile)) {
1855                     return;
1856                 }

1858                 if (libfile != null && loadLibrary0(fromClass, libfile)) {
1859                     return;
1860                 }
1861             }
1862         }

1864         throw new UnsatisfiedLinkError("no " + name + " in java.library.path");

1869     private static boolean loadLibrary0(Class<?> fromClass, final File file) {
1870         // Check to see if we're attempting to access a static library
1871         String name = findBuiltinLib(file.getName());
1872         boolean isBuiltin = (name != null);
1873         if (!isBuiltin) {
1874             boolean exists = AccessController.doPrivileged(
1875                 new PrivilegedAction<Object>() {
1876                     public Object run() {
1877                         return file.exists() ? Boolean.TRUE : null;
1878                     }})
1879                 != null;
1880             if (!exists) {
1881                 return false;
1882             }
1883             try {
1884                 name = file.getCanonicalPath();
1885             } catch (IOException e) {
1886                 return false;
1887             }
1888         }
1889         ClassLoader loader =
1890             (fromClass == null) ? null : fromClass.getClassLoader();
1891         Vector<NativeLibrary> libs =
1892             loader != null ? loader.nativeLibraries : systemNativeLibraries;
1893         synchronized (libs) {
1894             int size = libs.size();
1895             for (int i = 0; i < size; i++) {
1896                 NativeLibrary lib = libs.elementAt(i);
1897                 if (name.equals(lib.name)) {
1898                     return true;
1899                 }
1900             }
1901 
1902             synchronized (loadedLibraryNames) {
1903                 if (loadedLibraryNames.contains(name)) {
1904                     throw new UnsatisfiedLinkError
1905                         ("Native Library " +
1906                          name +
1907                          " already loaded in another classloader");
1908                 }
1909                 /* If the library is being loaded (must be by the same thread,
1910                  * because Runtime.load and Runtime.loadLibrary are
1911                  * synchronous). The reason is can occur is that the JNI_OnLoad
1912                  * function can cause another loadLibrary invocation.
1913                  *
1914                  * Thus we can use a static stack to hold the list of libraries
1915                  * we are loading.
1916                  *
1917                  * If there is a pending load operation for the library, we
1918                  * immediately return success; otherwise, we raise
1919                  * UnsatisfiedLinkError.
1920                  */
1921                 int n = nativeLibraryContext.size();
1922                 for (int i = 0; i < n; i++) {
1923                     NativeLibrary lib = nativeLibraryContext.elementAt(i);
1924                     if (name.equals(lib.name)) {
1925                         if (loader == lib.fromClass.getClassLoader()) {
1926                             return true;
1927                         } else {
1928                             throw new UnsatisfiedLinkError
1929                                 ("Native Library " +
1930                                  name +
1931                                  " is being loaded in another classloader");
1932                         }
1933                     }
1934                 }
1935                 NativeLibrary lib = new NativeLibrary(fromClass, name, isBuiltin);
1936                 nativeLibraryContext.push(lib);
1937                 try {
1938                     lib.load(name, isBuiltin);
1939                 } finally {
1940                     nativeLibraryContext.pop();
1941                 }
1942                 if (lib.loaded) {
1943                     loadedLibraryNames.addElement(name);
1944                     libs.addElement(lib);
1945                     return true;
1946                 }
1947                 return false;
1948             }
1949         }
1950     }
