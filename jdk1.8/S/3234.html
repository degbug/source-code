  31 /**
  32  * The Action interface provides a useful extension to the
  33  * ActionListener
  34  * interface in cases where the same functionality may be accessed by
  35  * several controls.
  36  * 
  37  * In addition to the actionPerformed method defined by the
  38  * ActionListener interface, this interface allows the
  39  * application to define, in a single place:
  40  * 
    
  41  * 
  • One or more text strings that describe the function. These strings
    • 
  42  *     can be used, for example, to display the flyover text for a button
  43  *     or to set the text in a menu item.
  44  * 
  • One or more icons that depict the function. These icons can be used
    • 
  45  *     for the images in a menu control, or for composite entries in a more
  46  *     sophisticated user interface.
  47  * 
  • The enabled/disabled state of the functionality. Instead of having
    • 
  48  *     to separately disable the menu item and the toolbar button, the
  49  *     application can disable the function that implements this interface.
  50  *     All components which are registered as listeners for the state change
  51  *     then know to disable event generation for that item and to modify the
  52  *     display accordingly.
  53  * 
  54  * 
  55  * This interface can be added to an existing class or used to create an
  56  * adapter (typically, by subclassing AbstractAction).
  57  * The Action object
  58  * can then be added to multiple Action-aware containers
  59  * and connected to Action-capable
  60  * components. The GUI controls can then be activated or
  61  * deactivated all at once by invoking the Action object's
  62  * setEnabled method.
  63  * 
  64  * Note that Action implementations tend to be more expensive
  65  * in terms of storage than a typical ActionListener,
  66  * which does not offer the benefits of centralized control of
  67  * functionality and broadcast of property changes.  For this reason,
  68  * you should take care to only use Actions where their benefits
  69  * are desired, and use simple ActionListeners elsewhere.
  70  * 
    
  71  *
  72  * 
    Swing Components Supporting Action
    
  73  * 
  74  * Many of Swing's components have an Action property.  When
  75  * an Action is set on a component, the following things
  76  * happen:
  77  * 
      
  78  * 
    • The Action is added as an ActionListener to
      • 
  79  *     the component.
  80  * 
    • The component configures some of its properties to match the
      • 
  81  *      Action.
  82  * 
    • The component installs a PropertyChangeListener on the
      • 
  83  *     Action so that the component can change its properties
  84  *     to reflect changes in the Action's properties.
  85  * 
  86  * 
  87  * The following table describes the properties used by
  88  * Swing components that support Actions.
  89  * In the table, button refers to any
  90  * AbstractButton subclass, which includes not only
  91  * JButton but also classes such as
  92  * JMenuItem. Unless otherwise stated, a
  93  * null property value in an Action (or a
  94  * Action that is null) results in the
  95  * button's corresponding property being set to null.
  96  *
  97  * 
  98  *         summary="Supported Action properties">
  99  *  
 100  *    Component Property
 101  *    Components
 102  *    Action Key
 103  *    Notes
 104  *  
 105  *      enabled
 106  *      All
 107  *      The isEnabled method
 108  *       
 109  *

 111  *      All
 112  *      SHORT_DESCRIPTION

 116  *      All

 119  *

 121  *      All buttons
 122  *      MNEMONIC_KEY
 123  *      A null value or Action results in the
 124  *          button's mnemonic property being set to
 125  *          '\0'.
 126  *  
 127  *      text
 128  *      All buttons
 129  *      NAME
 130  *      If you do not want the text of the button to mirror that
 131  *          of the Action, set the property
 132  *          hideActionText to true.  If
 133  *          hideActionText is true, setting the
 134  *          Action changes the text of the button to
 135  *          null and any changes to NAME
 136  *          are ignored.  hideActionText is useful for

 153  *

 171  *      JToggleButton, JCheckBox,
 172  *          JRadioButton, JCheckBoxMenuItem and
 173  *          JRadioButtonMenuItem
 174  *      SELECTED_KEY
 175  *      Components that honor this property only use
 176  *          the value if it is {@code non-null}. For example, if
 177  *          you set an {@code Action} that has a {@code null}
 178  *          value for {@code SELECTED_KEY} on a {@code JToggleButton}, the
 179  *          {@code JToggleButton} will not update it's selected state in
 180  *          any way. Similarly, any time the {@code JToggleButton}'s
 181  *          selected state changes it will only set the value back on
 182  *          the {@code Action} if the {@code Action} has a {@code non-null}
 183  *          value for {@code SELECTED_KEY}.
 184  *          
      
 185  *          Components that honor this property keep their selected state
 186  *          in sync with this property. When the same {@code Action} is used
 187  *          with multiple components, all the components keep their selected
 188  *          state in sync with this property. Mutually exclusive
 189  *          buttons, such as {@code JToggleButton}s in a {@code ButtonGroup},
 190  *          force only one of the buttons to be selected. As such, do not
 191  *          use the same {@code Action} that defines a value for the
 192  *          {@code SELECTED_KEY} property with multiple mutually
 193  *          exclusive buttons.
 194  * 
 195  * 
 196  * JPopupMenu, JToolBar and JMenu
 197  * all provide convenience methods for creating a component and setting the
 198  * Action on the corresponding component.  Refer to each of
 199  * these classes for more information.
 200  * 
 201  * Action uses PropertyChangeListener to
 202  * inform listeners the Action has changed.  The beans
 203  * specification indicates that a null property name can
 204  * be used to indicate multiple values have changed.  By default Swing
 205  * components that take an Action do not handle such a
 206  * change.  To indicate that Swing should treat null
 207  * according to the beans specification set the system property
 208  * swing.actions.reconfigureOnNull to the String
 209  * value true.
 210  *

 252     /**

 269     /**
 270      * The key used for storing an Integer that corresponds to
 271      * one of the KeyEvent key codes.  The value is
 272      * commonly used to specify a mnemonic.  For example:
 273      * myAction.putValue(Action.MNEMONIC_KEY, KeyEvent.VK_A)
 274      * sets the mnemonic of myAction to 'a', while
 275      * myAction.putValue(Action.MNEMONIC_KEY, KeyEvent.getExtendedKeyCodeForChar('\u0444'))
 276      * sets the mnemonic of myAction to Cyrillic letter "Ef".
 277      *
 278      * @since 1.3
 279      */
 280     public static final String MNEMONIC_KEY="MnemonicKey";
 281 
 282     /**
 283      * The key used for storing a Boolean that corresponds
 284      * to the selected state.  This is typically used only for components
 285      * that have a meaningful selection state.  For example,
 286      * JRadioButton and JCheckBox make use of
 287      * this but instances of JMenu don't.
 288      * 
 289      * This property differs from the others in that it is both read
 290      * by the component and set by the component.  For example,
 291      * if an Action is attached to a JCheckBox
 292      * the selected state of the JCheckBox will be set from
 293      * that of the Action.  If the user clicks on the
 294      * JCheckBox the selected state of the JCheckBox
 295      * and the Action will both be updated.
 296      * 
 297      * Note: the value of this field is prefixed with 'Swing' to
 298      * avoid possible collisions with existing Actions.
 299      *
 300      * @since 1.6
 301      */
 302     public static final String SELECTED_KEY = "SwingSelectedKey";
 303 
 304     /**

 355      * Sets the enabled state of the Action.  When enabled,
 356      * any component associated with this object is active and

 359      * to listeners.
 360      *
 361      * @param  b true to enable this Action, false to disable it

 363     public void setEnabled(boolean b);
 364     /**
 365      * Returns the enabled state of the Action. When enabled,
 366      * any component associated with this object is active and

 369      * @return true if this Action is enabled
 370      */
 371     public boolean isEnabled();
 372 
 373     /**

 379      * @param listener  a PropertyChangeListener object
