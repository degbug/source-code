  23 /**
  24  * Notations are how the Document Type Description (DTD) records hints
  25  * about the format of an XML "unparsed entity" -- in other words,
  26  * non-XML data bound to this document type, which some applications
  27  * may wish to consult when manipulating the document. A Notation
  28  * represents a name-value pair, with its nodeName being set to the
  29  * declared name of the notation.
  30  * <P>
  31  * Notations are also used to formally declare the "targets" of
  32  * Processing Instructions.
  33  * <P>
  34  * Note that the Notation's data is non-DOM information; the DOM only
  35  * records what and where it is.
  36  * <P>
  37  * See the XML 1.0 spec, sections 4.7 and 2.6, for more info.
  38  * <P>
  39  * Level 1 of the DOM does not support editing Notation contents.
  40  *
  41  * @xerces.internal
  42  *
  43  * @since  PR-DOM-Level-1-19980818.
