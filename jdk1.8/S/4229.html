  28 /**
  29  * An interface to be implemented by objects that can determine the
  30  * settings of an IIOParam object, either by putting up a
  31  * GUI to obtain values from a user, or by other means.  This
  32  * interface merely specifies a generic activate method
  33  * that invokes the controller, without regard for how the controller
  34  * obtains values (i.e., whether the controller puts up a GUI
  35  * or merely computes a set of values is irrelevant to this
  36  * interface).
  37  *
  38  *  Within the activate method, a controller obtains
  39  * initial values by querying the IIOParam object's
  40  * get methods, modifies values by whatever means, then
  41  * invokes the IIOParam object's set methods
  42  * to modify the appropriate settings.  Normally, these
  43  * set methods will be invoked all at once at a final
  44  * commit in order that a cancel operation not disturb existing
  45  * values.  In general, applications may expect that when the
  46  * activate method returns true, the
  47  * IIOParam object is ready for use in a read or write
  48  * operation.
  49  *
  50  *  Vendors may choose to provide GUIs for the
  51  * IIOParam subclasses they define for a particular
  52  * plug-in.  These can be set up as default controllers in the
  53  * corresponding IIOParam subclasses.
  54  *
  55  *  Applications may override any default GUIs and provide their
  56  * own controllers embedded in their own framework.  All that is
  57  * required is that theactivate method behave modally
  58  * (not returning until either cancelled or committed), though it need
  59  * not put up an explicitly modal dialog.  Such a non-modal GUI
  60  * component would be coded roughly as follows:
  61  *
  62  * 

  63  * 

  64  * class MyGUI extends SomeComponent implements IIOParamController {
  65  *
  66  *    public MyGUI() {
  67  *        // ...
  68  *        setEnabled(false);
  69  *    }
  70  *
  71  *    public boolean activate(IIOParam param) {
  72  *        // disable other components if desired
  73  *        setEnabled(true);
  74  *        // go to sleep until either cancelled or committed
  75  *        boolean ret = false;
  76  *        if (!cancelled) {
  77  *            // set values on param
  78  *            ret = true;
  79  *        }
  80  *        setEnabled(false);
  81  *        // enable any components disabled above
  82  *        return ret;
  83  *    }
  84  * 
  85  *
  86  *  Alternatively, an algorithmic process such as a database lookup
  87  * or the parsing of a command line could be used as a controller, in
  88  * which case the activate method would simply look up or
  89  * compute the settings, call the IIOParam.setXXX
  90  * methods, and return true.
  91  *
  92  * @see IIOParam#setController
  93  * @see IIOParam#getController
  94  * @see IIOParam#getDefaultController
  95  * @see IIOParam#hasController
  96  * @see IIOParam#activateController
  97  *
  98  */
  99 public interface IIOParamController {
 100 
 101     /**
 102      * Activates the controller.  If true is returned,
 103      * all settings in the IIOParam object should be
 104      * ready for use in a read or write operation.  If
 105      * false is returned, no settings in the
 106      * IIOParam object will be disturbed (i.e.,
 107      * the user canceled the operation).
 108      *
 109      * @param param the IIOParam object to be modified.
 110      *
 111      * @return true if the IIOParam has been
 112      * modified, false otherwise.
 113      *
 114      * @exception IllegalArgumentException if param is
 115      * null or is not an instance of the correct class.
 116      */
 117     boolean activate(IIOParam param);
 118 }
