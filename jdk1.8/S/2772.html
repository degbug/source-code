  41 /**
  42  * A lightweight container used behind the scenes by
  43  * JFrame, JDialog, JWindow,
  44  * JApplet, and JInternalFrame.
  45  * For task-oriented information on functionality provided by root panes
  46  * see How to Use Root Panes,
  47  * a section in The Java Tutorial.
  48  *
  49  * 
  50  * The following image shows the relationships between
  51  * the classes that use root panes.
  52  * 

  53  * alt="The following text describes this graphic."
  54  * HEIGHT=484 WIDTH=629>
  55  * The "heavyweight" components (those that delegate to a peer, or native
  56  * component on the host system) are shown with a darker, heavier box. The four
  57  * heavyweight JFC/Swing containers (JFrame, JDialog,
  58  * JWindow, and JApplet) are
  59  * shown in relation to the AWT classes they extend.
  60  * These four components are the
  61  * only heavyweight containers in the Swing library. The lightweight container
  62  * JInternalFrame is also shown.
  63  * All five of these JFC/Swing containers implement the
  64  * RootPaneContainer interface,
  65  * and they all delegate their operations to a
  66  * JRootPane (shown with a little "handle" on top).
  67  * 

  68  * Note: The JComponent method getRootPane
  69  * can be used to obtain the JRootPane that contains
  70  * a given component.
  71  * 
  72  * 

  73  * 
  74  * 
  75  * 
  76  * alt="The following text describes this graphic." HEIGHT=386 WIDTH=349>
  77  * 
  78  * 
  79  * 
  80  * The diagram at right shows the structure of a JRootPane.
  81  * A JRootpane is made up of a glassPane,
  82  * an optional menuBar, and a contentPane.
  83  * (The JLayeredPane manages the menuBar
  84  * and the contentPane.)
  85  * The glassPane sits over the top of everything,
  86  * where it is in a position to intercept mouse movements.
  87  * Since the glassPane (like the contentPane)
  88  * can be an arbitrary component, it is also possible to set up the
  89  * glassPane for drawing. Lines and images on the
  90  * glassPane can then range
  91  * over the frames underneath without being limited by their boundaries.
  92  * 
  93  * Although the menuBar component is optional,
  94  * the layeredPane, contentPane,
  95  * and glassPane always exist.
  96  * Attempting to set them to null generates an exception.
  97  * 
  98  * To add components to the JRootPane (other than the
  99  * optional menu bar), you add the object to the contentPane
 100  * of the JRootPane, like this:
 101  * 

 102  *       rootPane.getContentPane().add(child);
 103  * 
 104  * The same principle holds true for setting layout managers, removing
 105  * components, listing children, etc. All these methods are invoked on
 106  * the contentPane instead of on the JRootPane.
 107  * 

 108  * Note: The default layout manager for the contentPane is
 109  *  a BorderLayout manager. However, the JRootPane
 110  *  uses a custom LayoutManager.
 111  *  So, when you want to change the layout manager for the components you added
 112  *  to a JRootPane, be sure to use code like this:
 113  * 

 114  *    rootPane.getContentPane().setLayout(new BoxLayout());
 115  * 
 116  * If a JMenuBar component is set on the JRootPane,
 117  * it is positioned along the upper edge of the frame.
 118  * The contentPane is adjusted in location and size to
 119  * fill the remaining area.
 120  * (The JMenuBar and the contentPane are added to the
 121  * layeredPane component at the
 122  * JLayeredPane.FRAME_CONTENT_LAYER layer.)
 123  * 
 124  * The layeredPane is the parent of all children in the
 125  * JRootPane -- both as the direct parent of the menu and
 126  * the grandparent of all components added to the contentPane.
 127  * It is an instance of JLayeredPane,
 128  * which provides the ability to add components at several layers.
 129  * This capability is very useful when working with menu popups,
 130  * dialog boxes, and dragging -- situations in which you need to place
 131  * a component on top of all other components in the pane.
 132  * 
 133  * The glassPane sits on top of all other components in the
 134  * JRootPane.
 135  * That provides a convenient place to draw above all other components,
 136  * and makes it possible to intercept mouse events,
 137  * which is useful both for dragging and for drawing.
 138  * Developers can use setVisible on the glassPane
 139  * to control when the glassPane displays over the other children.
 140  * By default the glassPane is not visible.
 141  * 
 142  * The custom LayoutManager used by JRootPane
 143  * ensures that:
 144  * 
    
 145  * 
  1. The glassPane fills the entire viewable
    2. 
 146  *     area of the JRootPane (bounds - insets).
 147  * 
  2. The layeredPane fills the entire viewable area of the
    3. 
 148  *     JRootPane. (bounds - insets)
 149  * 
  3. The menuBar is positioned at the upper edge of the
    4. 
 150  *     layeredPane.
 151  * 
  4. The contentPane fills the entire viewable area,
    5. 
 152  *     minus the menuBar, if present.
 153  * 
 154  * Any other views in the JRootPane view hierarchy are ignored.
 155  * 
 156  * If you replace the LayoutManager of the JRootPane,
 157  * you are responsible for managing all of these views.
 158  * So ordinarily you will want to be sure that you
 159  * change the layout manager for the contentPane rather than
 160  * for the JRootPane itself!
 161  * 
 162  * The painting architecture of Swing requires an opaque
 163  * JComponent
 164  * to exist in the containment hierarchy above all other components. This is
 165  * typically provided by way of the content pane. If you replace the content
 166  * pane, it is recommended that you make the content pane opaque
 167  * by way of setOpaque(true). Additionally, if the content pane
 168  * overrides paintComponent, it
 169  * will need to completely fill in the background in an opaque color in
 170  * paintComponent.
 171  * 
 172  * Warning: Swing is not thread safe. For more
 173  * information see 
 174  * href="package-summary.html#threading">Swing's Threading
 175  * Policy.
 176  * 
 177  * Warning:
 178  * Serialized objects of this class will not be compatible with
 179  * future Swing releases. The current serialization support is
 180  * appropriate for short term storage or RMI between applications running
 181  * the same version of Swing.  As of 1.4, support for long term storage
 182  * of all JavaBeans™
 183  * has been added to the java.beans package.
 184  * Please see {@link java.beans.XMLEncoder}.
 185  *
 186  * @see JLayeredPane
 187  * @see JMenuBar
 188  * @see JWindow
 189  * @see JFrame
 190  * @see JDialog
 191  * @see JApplet
 192  * @see JInternalFrame
 193  * @see JComponent
 194  * @see BoxLayout
 195  *
 196  * @see 
 197  * Mixing Heavy and Light Components
 198  *
 199  * @author David Kloba

 667      * ever the root pane needs to grab input events.
 668      * 
 669      * The new glass pane's visibility is changed to match that of
 670      * the current glass pane.  An implication of this is that care
 671      * must be taken when you want to replace the glass pane and
 672      * make it visible.  Either of the following will work:
 673      * 
    
 674      *   root.setGlassPane(newGlassPane);
 675      *   newGlassPane.setVisible(true);
 676      * 
 677      * or:
 678      * 
    
 679      *   root.getGlassPane().setVisible(true);
 680      *   root.setGlassPane(newGlassPane);
 681      * 
 682      *
 683      * @param glass the Component to use as the glass pane
 684      *              for this JRootPane
 685      * @exception NullPointerException if the glass parameter is
 686      *          null
 687      */
 688     public void setGlassPane(Component glass) {
 689         if (glass == null) {
 690             throw new NullPointerException("glassPane cannot be set to null.");
 691         }
 692 
 693         AWTAccessor.getComponentAccessor().setMixingCutoutShape(glass,
 694                 new Rectangle());
 695 
 696         boolean visible = false;
 697         if (glassPane != null && glassPane.getParent() == this) {
 698             this.remove(glassPane);
 699             visible = glassPane.isVisible();
 700         }
 701 
 702         glass.setVisible(visible);
 703         glassPane = glass;
 704         this.add(glassPane, 0);
 705         if (visible) {
 706             repaint();
 707         }
 708     }

 772      * The default button is the button which will be activated

 790         JButton oldDefault = this.defaultButton;
 791 
 792         if (oldDefault != defaultButton) {
 793             this.defaultButton = defaultButton;
 794

 796                 oldDefault.repaint();
 797             }
 798             if (defaultButton != null) {
 799                 defaultButton.repaint();
 800             }
 801         }
 802 
 803         firePropertyChange("defaultButton", oldDefault, defaultButton);
 804     }

 892     /**
 893      * A custom layout manager that is responsible for the layout of
 894      * layeredPane, glassPane, and menuBar.
 895      * 
 896      * Warning:
 897      * Serialized objects of this class will not be compatible with
 898      * future Swing releases. The current serialization support is
 899      * appropriate for short term storage or RMI between applications running
 900      * the same version of Swing.  As of 1.4, support for long term storage
 901      * of all JavaBeans™
 902      * has been added to the java.beans package.
 903      * Please see {@link java.beans.XMLEncoder}.
 904      */
 905     @SuppressWarnings("serial")
 906     protected class RootLayout implements LayoutManager2, Serializable
 907     {
 908         /**
 909          * Returns the amount of space the layout would like to have.
 910          *
 911          * @param parent the Container for which this layout manager
 912          * is being used
 913          * @return a Dimension object containing the layout's preferred size
 914          */
 915         public Dimension preferredLayoutSize(Container parent) {
 916             Dimension rd, mbd;
 917             Insets i = getInsets();
 918 
 919             if(contentPane != null) {
 920                 rd = contentPane.getPreferredSize();
 921             } else {
 922                 rd = parent.getSize();
 923             }
 924             if(menuBar != null && menuBar.isVisible()) {
 925                 mbd = menuBar.getPreferredSize();
 926             } else {
 927                 mbd = new Dimension(0, 0);
 928             }
 929             return new Dimension(Math.max(rd.width, mbd.width) + i.left + i.right,
 930                                         rd.height + mbd.height + i.top + i.bottom);
 931         }
 932 
 933         /**
 934          * Returns the minimum amount of space the layout needs.
 935          *
 936          * @param parent the Container for which this layout manager
 937          * is being used
 938          * @return a Dimension object containing the layout's minimum size
 939          */
 940         public Dimension minimumLayoutSize(Container parent) {
 941             Dimension rd, mbd;
 942             Insets i = getInsets();
 943             if(contentPane != null) {
 944                 rd = contentPane.getMinimumSize();
 945             } else {
 946                 rd = parent.getSize();
 947             }
 948             if(menuBar != null && menuBar.isVisible()) {
 949                 mbd = menuBar.getMinimumSize();
 950             } else {
 951                 mbd = new Dimension(0, 0);
 952             }
 953             return new Dimension(Math.max(rd.width, mbd.width) + i.left + i.right,
 954                         rd.height + mbd.height + i.top + i.bottom);
 955         }
 956 
 957         /**
 958          * Returns the maximum amount of space the layout can use.
 959          *
 960          * @param target the Container for which this layout manager
 961          * is being used
 962          * @return a Dimension object containing the layout's maximum size
 963          */
 964         public Dimension maximumLayoutSize(Container target) {
 965             Dimension rd, mbd;
 966             Insets i = getInsets();
 967             if(menuBar != null && menuBar.isVisible()) {
 968                 mbd = menuBar.getMaximumSize();
 969             } else {
 970                 mbd = new Dimension(0, 0);
 971             }
 972             if(contentPane != null) {
 973                 rd = contentPane.getMaximumSize();
 974             } else {
 975                 // This is silly, but should stop an overflow error
 976                 rd = new Dimension(Integer.MAX_VALUE,
 977                         Integer.MAX_VALUE - i.top - i.bottom - mbd.height - 1);
 978             }
 979             return new Dimension(Math.min(rd.width, mbd.width) + i.left + i.right,
 980                                          rd.height + mbd.height + i.top + i.bottom);
 981         }
 982 
 983         /**
 984          * Instructs the layout manager to perform the layout for the specified
 985          * container.
 986          *
 987          * @param parent the Container for which this layout manager
 988          * is being used
 989          */
 990         public void layoutContainer(Container parent) {
 991             Rectangle b = parent.getBounds();
 992             Insets i = getInsets();
 993             int contentY = 0;
 994             int w = b.width - i.right - i.left;
 995             int h = b.height - i.top - i.bottom;
 996 
 997             if(layeredPane != null) {
 998                 layeredPane.setBounds(i.left, i.top, w, h);
 999             }
1000             if(glassPane != null) {
1001                 glassPane.setBounds(i.left, i.top, w, h);
1002             }
1003             // Note: This is laying out the children in the layeredPane,
1004             // technically, these are not our children.
1005             if(menuBar != null && menuBar.isVisible()) {
1006                 Dimension mbd = menuBar.getPreferredSize();
1007                 menuBar.setBounds(0, 0, w, mbd.height);
1008                 contentY += mbd.height;
1009             }
1010             if(contentPane != null) {
1011                 contentPane.setBounds(0, contentY, w, h - contentY);
1012             }
1013         }
