  42 /**
  43  * A synchronization point at which threads can pair and swap elements
  44  * within pairs.  Each thread presents some object on entry to the
  45  * {@link #exchange exchange} method, matches with a partner thread,
  46  * and receives its partner's object on return.  An Exchanger may be
  47  * viewed as a bidirectional form of a {@link SynchronousQueue}.
  48  * Exchangers may be useful in applications such as genetic algorithms
  49  * and pipeline designs.
  50  *
  51  * Sample Usage:
  52  * Here are the highlights of a class that uses an {@code Exchanger}
  53  * to swap buffers between threads so that the thread filling the
  54  * buffer gets a freshly emptied one when it needs it, handing off the
  55  * filled one to the thread emptying the buffer.
  56  *  
 {@code

  57  * class FillAndEmpty {
  58  *   Exchanger exchanger = new Exchanger();
  59  *   DataBuffer initialEmptyBuffer = ... a made-up type
  60  *   DataBuffer initialFullBuffer = ...
  61  *
  62  *   class FillingLoop implements Runnable {
  63  *     public void run() {
  64  *       DataBuffer currentBuffer = initialEmptyBuffer;
  65  *       try {
  66  *         while (currentBuffer != null) {
  67  *           addToBuffer(currentBuffer);
  68  *           if (currentBuffer.isFull())
  69  *             currentBuffer = exchanger.exchange(currentBuffer);
  70  *         }
  71  *       } catch (InterruptedException ex) { ... handle ... }
  72  *     }
  73  *   }
  74  *
  75  *   class EmptyingLoop implements Runnable {
  76  *     public void run() {
  77  *       DataBuffer currentBuffer = initialFullBuffer;
  78  *       try {
  79  *         while (currentBuffer != null) {
  80  *           takeFromBuffer(currentBuffer);
  81  *           if (currentBuffer.isEmpty())
  82  *             currentBuffer = exchanger.exchange(currentBuffer);
  83  *         }
  84  *       } catch (InterruptedException ex) { ... handle ...}
  85  *     }
  86  *   }
  87  *
  88  *   void start() {
  89  *     new Thread(new FillingLoop()).start();
  90  *     new Thread(new EmptyingLoop()).start();
  91  *   }
  92  * }}
  93  *
  94  * Memory consistency effects: For each pair of threads that
  95  * successfully exchange objects via an {@code Exchanger}, actions
  96  * prior to the {@code exchange()} in each thread
  97  * happen-before
  98  * those subsequent to a return from the corresponding {@code exchange()}
  99  * in the other thread.
 100  *
 101  * @since 1.5
 102  * @author Doug Lea and Bill Scherer and Michael Scott
 103  * @param  The type of objects that may be exchanged
 104  */
 105 public class Exchanger<V> {

 107     /*
 108      * Overview: The core algorithm is, for an exchange "slot",
 109      * and a participant (caller) with an item:
 110      *
 111      * for (;;) {
 112      *   if (slot is empty) {                       // offer
 113      *     place item in a Node;
 114      *     if (can CAS slot from empty to node) {
 115      *       wait for release;
 116      *       return matching item in node;
 117      *     }
 118      *   }
 119      *   else if (can CAS slot from node to empty) { // release
 120      *     get the item in node;
 121      *     set matching item in node;
 122      *     release waiting thread;
 123      *   }
 124      *   // else retry on CAS failure
 125      * }
 126      *
 127      * This is among the simplest forms of a "dual data structure" --
 128      * see Scott and Scherer's DISC 04 paper and
 129      * http://www.cs.rochester.edu/research/synchronization/pseudocode/duals.html
 130      *
 131      * This works great in principle. But in practice, like many
 132      * algorithms centered on atomic updates to a single location, it
 133      * scales horribly when there are more than a few participants
 134      * using the same Exchanger. So the implementation instead uses a
 135      * form of elimination arena, that spreads out this contention by
 136      * arranging that some threads typically use different slots,
 137      * while still ensuring that eventually, any two parties will be
 138      * able to exchange items. That is, we cannot completely partition
 139      * across threads, but instead give threads arena indices that
 140      * will on average grow under contention and shrink under lack of
 141      * contention. We approach this by defining the Nodes that we need
 142      * anyway as ThreadLocals, and include in them per-thread index
 143      * and related bookkeeping state. (We can safely reuse per-thread
 144      * nodes rather than creating them fresh each time because slots
 145      * alternate between pointing to a node vs null, so cannot
 146      * encounter ABA problems. However, we do need some care in
 147      * resetting them between uses.)
 148      *
 149      * Implementing an effective arena requires allocating a bunch of
 150      * space, so we only do so upon detecting contention (except on
 151      * uniprocessors, where they wouldn't help, so aren't used).
 152      * Otherwise, exchanges use the single-slot slotExchange method.
 153      * On contention, not only must the slots be in different
 154      * locations, but the locations must not encounter memory
 155      * contention due to being on the same cache line (or more
 156      * generally, the same coherence unit).  Because, as of this
 157      * writing, there is no way to determine cacheline size, we define
 158      * a value that is enough for common platforms.  Additionally,
 159      * extra care elsewhere is taken to avoid other false/unintended
 160      * sharing and to enhance locality, including adding padding (via
 161      * sun.misc.Contended) to Nodes, embedding "bound" as an Exchanger
 162      * field, and reworking some park/unpark mechanics compared to
 163      * LockSupport versions.
 164      *
 165      * The arena starts out with only one used slot. We expand the
 166      * effective arena size by tracking collisions; i.e., failed CASes
 167      * while trying to exchange. By nature of the above algorithm, the
 168      * only kinds of collision that reliably indicate contention are
 169      * when two attempted releases collide -- one of two attempted
 170      * offers can legitimately fail to CAS without indicating
 171      * contention by more than one other thread. (Note: it is possible
 172      * but not worthwhile to more precisely detect contention by
 173      * reading slot values after CAS failures.)  When a thread has
 174      * collided at each slot within the current arena bound, it tries
 175      * to expand the arena size by one. We track collisions within
 176      * bounds by using a version (sequence) number on the "bound"
 177      * field, and conservatively reset collision counts when a
 178      * participant notices that bound has been updated (in either
 179      * direction).
 180      *
 181      * The effective arena size is reduced (when there is more than
 182      * one slot) by giving up on waiting after a while and trying to
 183      * decrement the arena size on expiration. The value of "a while"
 184      * is an empirical matter.  We implement by piggybacking on the
 185      * use of spin->yield->block that is essential for reasonable
 186      * waiting performance anyway -- in a busy exchanger, offers are
 187      * usually almost immediately released, in which case context
 188      * switching on multiprocessors is extremely slow/wasteful.  Arena
 189      * waits just omit the blocking part, and instead cancel. The spin
 190      * count is empirically chosen to be a value that avoids blocking
 191      * 99% of the time under maximum sustained exchange rates on a
 192      * range of test machines. Spins and yields entail some limited
 193      * randomness (using a cheap xorshift) to avoid regular patterns
 194      * that can induce unproductive grow/shrink cycles. (Using a
 195      * pseudorandom also helps regularize spin cycle duration by
 196      * making branches unpredictable.)  Also, during an offer, a
 197      * waiter can "know" that it will be released when its slot has
 198      * changed, but cannot yet proceed until match is set.  In the
 199      * mean time it cannot cancel the offer, so instead spins/yields.
 200      * Note: It is possible to avoid this secondary check by changing
 201      * the linearization point to be a CAS of the match field (as done
 202      * in one case in the Scott & Scherer DISC paper), which also
 203      * increases asynchrony a bit, at the expense of poorer collision
 204      * detection and inability to always reuse per-thread nodes. So
 205      * the current scheme is typically a better tradeoff.
 206      *
 207      * On collisions, indices traverse the arena cyclically in reverse
 208      * order, restarting at the maximum index (which will tend to be
 209      * sparsest) when bounds change. (On expirations, indices instead
 210      * are halved until reaching 0.) It is possible (and has been
 211      * tried) to use randomized, prime-value-stepped, or double-hash
 212      * style traversal instead of simple cyclic traversal to reduce
 213      * bunching.  But empirically, whatever benefits these may have
 214      * don't overcome their added overhead: We are managing operations
 215      * that occur very quickly unless there is sustained contention,
 216      * so simpler/faster control policies work better than more
 217      * accurate but slower ones.
 218      *
 219      * Because we use expiration for arena size control, we cannot
 220      * throw TimeoutExceptions in the timed version of the public
 221      * exchange method until the arena size has shrunken to zero (or
 222      * the arena isn't enabled). This may delay response to timeout
 223      * but is still within spec.
 224      *
 225      * Essentially all of the implementation is in methods
 226      * slotExchange and arenaExchange. These have similar overall
 227      * structure, but differ in too many details to combine. The
 228      * slotExchange method uses the single Exchanger field "slot"
 229      * rather than arena array elements. However, it still needs
 230      * minimal collision detection to trigger arena construction.
 231      * (The messiest part is making sure interrupt status and
 232      * InterruptedExceptions come out right during transitions when
 233      * both methods may be called. This is done by using null return
 234      * as a sentinel to recheck interrupt status.)
 235      *
 236      * As is too common in this sort of code, methods are monolithic
 237      * because most of the logic relies on reads of fields that are
 238      * maintained as local variables so can't be nicely factored --
 239      * mainly, here, bulky spin->yield->block/cancel code), and
 240      * heavily dependent on intrinsics (Unsafe) to use inlined
 241      * embedded CAS and related memory access operations (that tend
 242      * not to be as readily inlined by dynamic compilers when they are
 243      * hidden behind other methods that would more nicely name and
 244      * encapsulate the intended effects). This includes the use of
 245      * putOrderedX to clear fields of the per-thread Nodes between
 246      * uses. Note that field Node.item is not declared as volatile
 247      * even though it is read by releasing threads, because they only
 248      * do so after CAS operations that must precede access, and all
 249      * uses by the owning thread are otherwise acceptably ordered by
 250      * other operations. (Because the actual points of atomicity are
 251      * slot CASes, it would also be legal for the write to Node.match
 252      * in a release to be weaker than a full volatile write. However,
 253      * this is not done because it could allow further postponement of
 254      * the write, delaying progress.)
 255      */

 352     /**
 353      * Exchange function when arenas enabled. See above for explanation.
 354      *
 355      * @param item the (non-null) item to exchange
 356      * @param timed true if the wait is timed
 357      * @param ns if timed, the maximum wait time, else 0L
 358      * @return the other thread's item; or null if interrupted; or
 359      * TIMED_OUT if timed and timed out
 360      */
 361     private final Object arenaExchange(Object item, boolean timed, long ns) {
 362         Node[] a = arena;
 363         Node p = participant.get();
 364         for (int i = p.index;;) {                      // access slot at i
 365             int b, m, c; long j;                       // j is raw array offset
 366             Node q = (Node)U.getObjectVolatile(a, j = (i << ASHIFT) + ABASE);
 367             if (q != null && U.compareAndSwapObject(a, j, q, null)) {
 368                 Object v = q.item;                     // release
 369                 q.match = item;
 370                 Thread w = q.parked;
 371                 if (w != null)
 372                     U.unpark(w);
 373                 return v;
 374             }
 375             else if (i <= (m = (b = bound) & MMASK) && q == null) {
 376                 p.item = item;                         // offer
 377                 if (U.compareAndSwapObject(a, j, null, p)) {
 378                     long end = (timed && m == 0) ? System.nanoTime() + ns : 0L;
 379                     Thread t = Thread.currentThread(); // wait
 380                     for (int h = p.hash, spins = SPINS;;) {
 381                         Object v = p.match;
 382                         if (v != null) {
 383                             U.putOrderedObject(p, MATCH, null);
 384                             p.item = null;             // clear for next use
 385                             p.hash = h;
 386                             return v;
 387                         }
 388                         else if (spins > 0) {
 389                             h ^= h << 1; h ^= h >>> 3; h ^= h << 10; // xorshift
 390                             if (h == 0)                // initialize hash
 391                                 h = SPINS | (int)t.getId();
 392                             else if (h < 0 &&          // approx 50% true
 393                                      (--spins & ((SPINS >>> 1) - 1)) == 0)
 394                                 Thread.yield();        // two yields per wait
 395                         }
 396                         else if (U.getObjectVolatile(a, j) != p)
 397                             spins = SPINS;       // releaser hasn't set match yet
 398                         else if (!t.isInterrupted() && m == 0 &&
 399                                  (!timed ||
 400                                   (ns = end - System.nanoTime()) > 0L)) {
 401                             U.putObject(t, BLOCKER, this); // emulate LockSupport
 402                             p.parked = t;              // minimize window
 403                             if (U.getObjectVolatile(a, j) == p)
 404                                 U.park(false, ns);
 405                             p.parked = null;
 406                             U.putObject(t, BLOCKER, null);
 407                         }
 408                         else if (U.getObjectVolatile(a, j) == p &&
 409                                  U.compareAndSwapObject(a, j, p, null)) {
 410                             if (m != 0)                // try to shrink
 411                                 U.compareAndSwapInt(this, BOUND, b, b + SEQ - 1);
 412                             p.item = null;
 413                             p.hash = h;
 414                             i = p.index >>>= 1;        // descend
 415                             if (Thread.interrupted())
 416                                 return null;
 417                             if (timed && m == 0 && ns <= 0L)
 418                                 return TIMED_OUT;
 419                             break;                     // expired; restart
 420                         }
 421                     }
 422                 }
 423                 else
 424                     p.item = null;                     // clear offer
 425             }
 426             else {
 427                 if (p.bound != b) {                    // stale; reset
 428                     p.bound = b;
 429                     p.collides = 0;
 430                     i = (i != m || m == 0) ? m : m - 1;
 431                 }
 432                 else if ((c = p.collides) < m || m == FULL ||
 433                          !U.compareAndSwapInt(this, BOUND, b, b + SEQ + 1)) {
 434                     p.collides = c + 1;
 435                     i = (i == 0) ? m : i - 1;          // cyclically traverse
 436                 }
 437                 else
 438                     i = m + 1;                         // grow
 439                 p.index = i;
 440             }
 441         }
 442     }

 444     /**
 445      * Exchange function used until arenas enabled. See above for explanation.
 446      *
 447      * @param item the item to exchange
 448      * @param timed true if the wait is timed
 449      * @param ns if timed, the maximum wait time, else 0L
 450      * @return the other thread's item; or null if either the arena
 451      * was enabled or the thread was interrupted before completion; or
 452      * TIMED_OUT if timed and timed out
 453      */
 454     private final Object slotExchange(Object item, boolean timed, long ns) {
 455         Node p = participant.get();
 456         Thread t = Thread.currentThread();
 457         if (t.isInterrupted()) // preserve interrupt status so caller can recheck
 458             return null;
 459 
 460         for (Node q;;) {
 461             if ((q = slot) != null) {
 462                 if (U.compareAndSwapObject(this, SLOT, q, null)) {
 463                     Object v = q.item;
 464                     q.match = item;
 465                     Thread w = q.parked;
 466                     if (w != null)
 467                         U.unpark(w);
 468                     return v;
 469                 }
 470                 // create arena on contention, but continue until slot null
 471                 if (NCPU > 1 && bound == 0 &&
 472                     U.compareAndSwapInt(this, BOUND, 0, SEQ))
 473                     arena = new Node[(FULL + 2) << ASHIFT];
 474             }
 475             else if (arena != null)
 476                 return null; // caller must reroute to arenaExchange
 477             else {
 478                 p.item = item;
 479                 if (U.compareAndSwapObject(this, SLOT, null, p))
 480                     break;
 481                 p.item = null;
 482             }
 483         }
 484 
 485         // await release
 486         int h = p.hash;
 487         long end = timed ? System.nanoTime() + ns : 0L;
 488         int spins = (NCPU > 1) ? SPINS : 1;
 489         Object v;
 490         while ((v = p.match) == null) {
 491             if (spins > 0) {
 492                 h ^= h << 1; h ^= h >>> 3; h ^= h << 10;
 493                 if (h == 0)
 494                     h = SPINS | (int)t.getId();
 495                 else if (h < 0 && (--spins & ((SPINS >>> 1) - 1)) == 0)
 496                     Thread.yield();
 497             }
 498             else if (slot != p)
 499                 spins = SPINS;
 500             else if (!t.isInterrupted() && arena == null &&
 501                      (!timed || (ns = end - System.nanoTime()) > 0L)) {
 502                 U.putObject(t, BLOCKER, this);
 503                 p.parked = t;
 504                 if (slot == p)
 505                     U.park(false, ns);
 506                 p.parked = null;
 507                 U.putObject(t, BLOCKER, null);
 508             }
 509             else if (U.compareAndSwapObject(this, SLOT, p, null)) {
 510                 v = timed && ns <= 0L && !t.isInterrupted() ? TIMED_OUT : null;
 511                 break;
 512             }
 513         }
 514         U.putOrderedObject(p, MATCH, null);
 515         p.item = null;
 516         p.hash = h;
 517         return v;
 518     }

 527     /**
 528      * Waits for another thread to arrive at this exchange point (unless
 529      * the current thread is {@linkplain Thread#interrupt interrupted}),
 530      * and then transfers the given object to it, receiving its object
 531      * in return.
 532      *
 533      * If another thread is already waiting at the exchange point then
 534      * it is resumed for thread scheduling purposes and receives the object
 535      * passed in by the current thread.  The current thread returns immediately,
 536      * receiving the object passed to the exchange by that other thread.
 537      *
 538      * If no other thread is already waiting at the exchange then the
 539      * current thread is disabled for thread scheduling purposes and lies
 540      * dormant until one of two things happens:
 541      * 
    
 542      * 
  • Some other thread enters the exchange; or
    • 
 543      * 
  • Some other thread {@linkplain Thread#interrupt interrupts}
    • 
 544      * the current thread.
 545      * 
 546      * If the current thread:
 547      * 
      
 548      * 
    • has its interrupted status set on entry to this method; or
      • 
 549      * 
    • is {@linkplain Thread#interrupt interrupted} while waiting
      • 
 550      * for the exchange,
 551      * 
 552      * then {@link InterruptedException} is thrown and the current thread's
 553      * interrupted status is cleared.
 554      *
 555      * @param x the object to exchange
 556      * @return the object provided by the other thread
 557      * @throws InterruptedException if the current thread was
 558      *         interrupted while waiting
 559      */
 560     @SuppressWarnings("unchecked")
 561     public V exchange(V x) throws InterruptedException {
 562         Object v;
 563         Object item = (x == null) ? NULL_ITEM : x; // translate null args
 564         if ((arena != null ||
 565              (v = slotExchange(item, false, 0L)) == null) &&
 566             ((Thread.interrupted() || // disambiguates null return
 567               (v = arenaExchange(item, false, 0L)) == null)))
 568             throw new InterruptedException();
 569         return (v == NULL_ITEM) ? null : (V)v;
 570     }

 572     /**
 573      * Waits for another thread to arrive at this exchange point (unless
 574      * the current thread is {@linkplain Thread#interrupt interrupted} or
 575      * the specified waiting time elapses), and then transfers the given
 576      * object to it, receiving its object in return.
 577      *
 578      * If another thread is already waiting at the exchange point then
 579      * it is resumed for thread scheduling purposes and receives the object
 580      * passed in by the current thread.  The current thread returns immediately,
 581      * receiving the object passed to the exchange by that other thread.
 582      *
 583      * If no other thread is already waiting at the exchange then the
 584      * current thread is disabled for thread scheduling purposes and lies
 585      * dormant until one of three things happens:
 586      * 
        
 587      * 
      • Some other thread enters the exchange; or
        • 
 588      * 
      • Some other thread {@linkplain Thread#interrupt interrupts}
        • 
 589      * the current thread; or
 590      * 
      • The specified waiting time elapses.
        • 
 591      * 
 592      * If the current thread:
 593      * 
          
 594      * 
        • has its interrupted status set on entry to this method; or
          • 
 595      * 
        • is {@linkplain Thread#interrupt interrupted} while waiting
          • 
 596      * for the exchange,
 597      * 
 598      * then {@link InterruptedException} is thrown and the current thread's
 599      * interrupted status is cleared.
 600      *
 601      * If the specified waiting time elapses then {@link
 602      * TimeoutException} is thrown.  If the time is less than or equal
 603      * to zero, the method will not wait at all.
 604      *
 605      * @param x the object to exchange
 606      * @param timeout the maximum time to wait
 607      * @param unit the time unit of the {@code timeout} argument
 608      * @return the object provided by the other thread
 609      * @throws InterruptedException if the current thread was
 610      *         interrupted while waiting
 611      * @throws TimeoutException if the specified waiting time elapses
 612      *         before another thread enters the exchange
 613      */
 614     @SuppressWarnings("unchecked")
 615     public V exchange(V x, long timeout, TimeUnit unit)
 616         throws InterruptedException, TimeoutException {
 617         Object v;
 618         Object item = (x == null) ? NULL_ITEM : x;
 619         long ns = unit.toNanos(timeout);
 620         if ((arena != null ||
 621              (v = slotExchange(item, true, ns)) == null) &&
 622             ((Thread.interrupted() ||
 623               (v = arenaExchange(item, true, ns)) == null)))
 624             throw new InterruptedException();
 625         if (v == TIMED_OUT)
 626             throw new TimeoutException();
 627         return (v == NULL_ITEM) ? null : (V)v;
 628     }
