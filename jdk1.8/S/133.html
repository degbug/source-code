  49 /**
  50  *  An interface to an object that is able to build, or augment, a DOM tree
  51  * from various input sources.
  52  *  LSParser provides an API for parsing XML and building the
  53  * corresponding DOM document structure. A LSParser instance
  54  * can be obtained by invoking the
  55  * DOMImplementationLS.createLSParser() method.
  56  *  As specified in [DOM Level 3 Core]
  57  * , when a document is first made available via the LSParser:
  58  * 
    
  59  * 
  •  there will
    • 
  60  * never be two adjacent nodes of type NODE_TEXT, and there will never be
  61  * empty text nodes.
  62  * 
  63  * 
  •  it is expected that the value and
    • 
  64  * nodeValue attributes of an Attr node initially
  65  * return the XML 1.0
  66  * normalized value. However, if the parameters "
  67  * validate-if-schema" and "
  68  * datatype-normalization" are set to true, depending on the attribute normalization
  69  * used, the attribute values may differ from the ones obtained by the XML
  70  * 1.0 attribute normalization. If the parameters "
  71  * datatype-normalization" is set to false, the XML 1.0 attribute normalization is
  72  * guaranteed to occur, and if the attributes list does not contain
  73  * namespace declarations, the attributes attribute on
  74  * Element node represents the property [attributes] defined in [XML Information Set]
  75  * .
  76  * 
  77  * 
  78  *  Asynchronous LSParser objects are expected to also
  79  * implement the events::EventTarget interface so that event
  80  * listeners can be registered on asynchronous LSParser
  81  * objects.
  82  *  Events supported by asynchronous LSParser objects are:
  83  * 
    
  84  * 
    load
    
  85  * 
    
  86  *  The LSParser finishes to load the document. See also the
  87  * definition of the LSLoadEvent interface. 
  88  * 
    progress
    
  89  * 
     The
    
  90  * LSParser signals progress as data is parsed.  This
  91  * specification does not attempt to define exactly when progress events
  92  * should be dispatched. That is intentionally left as
  93  * implementation-dependent. Here is one example of how an application might
  94  * dispatch progress events: Once the parser starts receiving data, a
  95  * progress event is dispatched to indicate that the parsing starts. From
  96  * there on, a progress event is dispatched for every 4096 bytes of data
  97  * that is received and processed. This is only one example, though, and
  98  * implementations can choose to dispatch progress events at any time while
  99  * parsing, or not dispatch them at all.  See also the definition of the
 100  * LSProgressEvent interface. 
 101  * 
 102  * 
    Note:  All events defined in this specification use the
    
 103  * namespace URI "http://www.w3.org/2002/DOMLS".
 104  *  While parsing an input source, errors are reported to the application
 105  * through the error handler (LSParser.domConfig's "
 106  * error-handler" parameter). This specification does in no way try to define all possible
 107  * errors that can occur while parsing XML, or any other markup, but some
 108  * common error cases are defined. The types (DOMError.type) of
 109  * errors and warnings defined by this specification are:
 110  * 
    
 111  * 
    
 112  * "check-character-normalization-failure" [error] 
 113  * 
     Raised if
    
 114  * the parameter "
 115  * check-character-normalization" is set to true and a string is encountered that fails normalization
 116  * checking. 
 117  * 
    "doctype-not-allowed" [fatal]
    
 118  * 
     Raised if the
    
 119  * configuration parameter "disallow-doctype" is set to true
 120  * and a doctype is encountered. 
 121  * 
    "no-input-specified" [fatal]
    
 122  * 
    
 123  * Raised when loading a document and no input is specified in the
 124  * LSInput object. 
 125  * 
    
 126  * "pi-base-uri-not-preserved" [warning]
 127  * 
     Raised if a processing
    
 128  * instruction is encountered in a location where the base URI of the
 129  * processing instruction can not be preserved.  One example of a case where
 130  * this warning will be raised is if the configuration parameter "
 131  * entities" is set to false and the following XML file is parsed:
 132  * 
    
 133  * <!DOCTYPE root [ <!ENTITY e SYSTEM 'subdir/myentity.ent' ]>
 134  * <root> &e; </root>
 135  *  And subdir/myentity.ent
 136  * contains:
 137  * 
    <one> <two/> </one> <?pi 3.14159?>
    
 138  * <more/>
 139  * 
 140  * 
    "unbound-prefix-in-entity" [warning]
    
 141  * 
     An
    
 142  * implementation dependent warning that may be raised if the configuration
 143  * parameter "
 144  * namespaces" is set to true and an unbound namespace prefix is
 145  * encountered in an entity's replacement text. Raising this warning is not
 146  * enforced since some existing parsers may not recognize unbound namespace
 147  * prefixes in the replacement text of entities. 
 148  * 
    
 149  * "unknown-character-denormalization" [fatal]
 150  * 
     Raised if the
    
 151  * configuration parameter "ignore-unknown-character-denormalizations" is
 152  * set to false and a character is encountered for which the
 153  * processor cannot determine the normalization properties. 
 154  * 
    
 155  * "unsupported-encoding" [fatal]
 156  * 
     Raised if an unsupported
    
 157  * encoding is encountered. 
 158  * 
    "unsupported-media-type" [fatal]
    
 159  * 
    
 160  * Raised if the configuration parameter "supported-media-types-only" is set
 161  * to true and an unsupported media type is encountered. 
 162  * 
 163  *  In addition to raising the defined errors and warnings, implementations
 164  * are expected to raise implementation specific errors and warnings for any
 165  * other error and warning cases such as IO errors (file not found,
 166  * permission denied,...), XML well-formedness errors, and so on.
 167  * See also the Document Object Model (DOM) Level 3 Load
 168 and Save Specification.
 169  */

 171     /**
 172      *  The DOMConfiguration object used when parsing an input
 173      * source. This DOMConfiguration is specific to the parse
 174      * operation. No parameter values from this DOMConfiguration
 175      *  object are passed automatically to the DOMConfiguration
 176      * object on the Document that is created, or used, by the
 177      * parse operation. The DOM application is responsible for passing any
 178      * needed parameter values from this DOMConfiguration
 179      * object to the DOMConfiguration object referenced by the
 180      * Document object.
 181      * 
     In addition to the parameters recognized in on the     
 182      * DOMConfiguration interface defined in [DOM Level 3 Core]
 183      * , the DOMConfiguration objects for LSParser
 184      * add or modify the following parameters:
 185      * 
    
 186      * 
    
 187      * "charset-overrides-xml-encoding"
 188      * 
    
 189      * 
    
 190      * 
    true
    
 191      * 
    [optional] (default) If a higher level protocol such as HTTP [IETF RFC 2616] provides an
    
 192      * indication of the character encoding of the input stream being
 193      * processed, that will override any encoding specified in the XML
 194      * declaration or the Text declaration (see also section 4.3.3,
 195      * "Character Encoding in Entities", in [XML 1.0]).
 196      * Explicitly setting an encoding in the LSInput overrides
 197      * any encoding from the protocol. 
 198      * 
    false
    
 199      * 
    [required] The parser ignores any character set encoding information from
    
 200      * higher-level protocols. 
 201      * 
 202      * 
    "disallow-doctype"
    
 203      * 
    
 204      * 
    
 205      * 
    
 206      * true
 207      * 
    [optional] Throw a fatal "doctype-not-allowed" error if a doctype node is found while parsing the document. This is
    
 208      * useful when dealing with things like SOAP envelopes where doctype
 209      * nodes are not allowed. 
 210      * 
    false
    
 211      * 
    [required] (default) Allow doctype nodes in the document. 
    
 212      * 
 213      * 
    
 214      * "ignore-unknown-character-denormalizations"
 215      * 
    
 216      * 
    
 217      * 
    
 218      * true
 219      * 
    [required] (default) If, while verifying full normalization when [XML 1.1] is
    
 220      * supported, a processor encounters characters for which it cannot
 221      * determine the normalization properties, then the processor will
 222      * ignore any possible denormalizations caused by these characters.
 223      * This parameter is ignored for [XML 1.0]. 
 224      * 
    
 225      * false
 226      * 
    [optional] Report an fatal "unknown-character-denormalization" error if a character is encountered for which the processor cannot
    
 227      * determine the normalization properties. 
 228      * 
 229      * 
    "infoset"
    
 230      * 
     See
    
 231      * the definition of DOMConfiguration for a description of
 232      * this parameter. Unlike in [DOM Level 3 Core]
 233      * , this parameter will default to true for
 234      * LSParser. 
 235      * 
    "namespaces"
    
 236      * 
    
 237      * 
    
 238      * 
    true
    
 239      * 
    [required] (default) Perform the namespace processing as defined in [XML Namespaces]
    
 240      *  and [XML Namespaces 1.1]
 241      * . 
 242      * 
    false
    
 243      * 
    [optional] Do not perform the namespace processing. 
    
 244      * 
 245      * 
    
 246      * "resource-resolver"
 247      * 
    [required] A reference to a LSResourceResolver object, or null. If
    
 248      * the value of this parameter is not null when an external resource
 249      * (such as an external XML entity or an XML schema location) is
 250      * encountered, the implementation will request that the
 251      * LSResourceResolver referenced in this parameter resolves
 252      * the resource. 
 253      * 
    "supported-media-types-only"
    
 254      * 
    
 255      * 
    
 256      * 
    
 257      * true
 258      * 
    [optional] Check that the media type of the parsed resource is a supported media
    
 259      * type. If an unsupported media type is encountered, a fatal error of
 260      * type "unsupported-media-type" will be raised. The media types defined in [IETF RFC 3023] must always
 261      * be accepted. 
 262      * 
    false
    
 263      * 
    [required] (default) Accept any media type. 
    
 264      * 
 265      * 
    "validate"
    
 266      * 
     See the definition of
    
 267      * DOMConfiguration for a description of this parameter.
 268      * Unlike in [DOM Level 3 Core]
 269      * , the processing of the internal subset is always accomplished, even
 270      * if this parameter is set to false. 
 271      * 
    
 272      * "validate-if-schema"
 273      * 
     See the definition of
    
 274      * DOMConfiguration for a description of this parameter.
 275      * Unlike in [DOM Level 3 Core]
 276      * , the processing of the internal subset is always accomplished, even
 277      * if this parameter is set to false. 
 278      * 
    
 279      * "well-formed"
 280      * 
     See the definition of
    
 281      * DOMConfiguration for a description of this parameter.
 282      * Unlike in [DOM Level 3 Core]
 283      * , this parameter cannot be set to false. 
 284      * 
 285      */

 407     /**
 408      *  Parse an XML fragment from a resource identified by a
 409      * LSInput and insert the content into an existing document
 410      * at the position specified with the context and
 411      * action arguments. When parsing the input stream, the
 412      * context node (or its parent, depending on where the result will be
 413      * inserted) is used for resolving unbound namespace prefixes. The
 414      * context node's ownerDocument node (or the node itself if
 415      * the node of type DOCUMENT_NODE) is used to resolve
 416      * default attributes and entity references.
 417      * 
     As the new data is inserted into the document, at least one    
 418      * mutation event is fired per new immediate child or sibling of the
 419      * context node.
 420      * 
     If the context node is a Document node and the action    
 421      * is ACTION_REPLACE_CHILDREN, then the document that is
 422      * passed as the context node will be changed such that its
 423      * xmlEncoding, documentURI,
 424      * xmlVersion, inputEncoding,
 425      * xmlStandalone, and all other such attributes are set to
 426      * what they would be set to if the input source was parsed using
 427      * LSParser.parse().
 428      * 
     This method is always synchronous, even if the    
 429      * LSParser is asynchronous (LSParser.async is
 430      * true).
 431      * 
     If an error occurs while parsing, the caller is notified through    
 432      * the ErrorHandler instance associated with the "
 433      * error-handler" parameter of the DOMConfiguration.
 434      * 
     When calling parseWithContext, the values of the    
 435      * following configuration parameters will be ignored and their default
 436      * values will always be used instead: "
 437      * validate", "
 438      * validate-if-schema", and "
 439      * element-content-whitespace". Other parameters will be treated normally, and the parser is expected
 440      * to call the LSParserFilter just as if a whole document
 441      * was parsed.
 442      * @param input  The LSInput from which the source document
 443      *   is to be read. The source document must be an XML fragment, i.e.
 444      *   anything except a complete XML document (except in the case where
 445      *   the context node of type DOCUMENT_NODE, and the action
 446      *   is ACTION_REPLACE_CHILDREN), a DOCTYPE (internal
 447      *   subset), entity declaration(s), notation declaration(s), or XML or
 448      *   text declaration(s).
 449      * @param contextArg  The node that is used as the context for the data
 450      *   that is being parsed. This node must be a Document
 451      *   node, a DocumentFragment node, or a node of a type
 452      *   that is allowed as a child of an Element node, e.g. it
 453      *   cannot be an Attribute node.
 454      * @param action  This parameter describes which action should be taken
 455      *   between the new set of nodes being inserted and the existing
 456      *   children of the context node. The set of possible actions is
 457      *   defined in ACTION_TYPES above.
 458      * @return  Return the node that is the result of the parse operation. If
 459      *   the result is more than one top-level node, the first one is
 460      *   returned.
 461      * @exception DOMException
 462      *   HIERARCHY_REQUEST_ERR: Raised if the content cannot replace, be
 463      *   inserted before, after, or as a child of the context node (see also
 464      *   Node.insertBefore or Node.replaceChild in [DOM Level 3 Core]
 465      *   ).
 466      *   
     NOT_SUPPORTED_ERR: Raised if the LSParser doesn't    
 467      *   support this method, or if the context node is of type
 468      *   Document and the DOM implementation doesn't support
 469      *   the replacement of the DocumentType child or
 470      *   Element child.
 471      *   
     NO_MODIFICATION_ALLOWED_ERR: Raised if the context node is a    
 472      *   read only node and the content is being appended to its child list,
 473      *   or if the parent node of the context node is read only node and the
 474      *   content is being inserted in its child list.
 475      *   
     INVALID_STATE_ERR: Raised if the LSParser.busy    
 476      *   attribute is true.
 477      * @exception LSException
 478      *    PARSE_ERR: Raised if the LSParser was unable to load
 479      *   the XML fragment. DOM applications should attach a
 480      *   DOMErrorHandler using the parameter "
 481      *   error-handler" if they wish to get details on the error.
 482      */
 483     public Node parseWithContext(LSInput input,
 484                                  Node contextArg,
 485                                  short action)
 486                                  throws DOMException, LSException;

 492      */
 493     public void abort();
