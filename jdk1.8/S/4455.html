  31 /**
  32  * Defines a framework that allows applications to use a manual decision tree
  33  * to decide what should be done when a synchronization conflict occurs.
  34  * Although it is not mandatory for
  35  * applications to resolve synchronization conflicts manually, this
  36  * framework provides the means to delegate to the application when conflicts
  37  * arise.
  38  * 
  39  * Note that a conflict is a situation where the RowSet object's original
  40  * values for a row do not match the values in the data source, which indicates that
  41  * the data source row has been modified since the last synchronization. Note also that
  42  * a RowSet object's original values are the values it had just prior to the
  43  * the last synchronization, which are not necessarily its initial values.
  44  *
  45  *
  46  * 
Description of a SyncResolver Object

  47  *
  48  * A SyncResolver object is a specialized RowSet object
  49  * that implements the SyncResolver interface.
  50  * It may operate as either a connected RowSet object (an
  51  * implementation of the JdbcRowSet interface) or a connected
  52  * RowSet object (an implementation of the
  53  * CachedRowSet interface or one of its subinterfaces). For information
  54  * on the subinterfaces, see the
  55  * javax.sql.rowset package
  56  * description. The reference implementation for SyncResolver implements
  57  * the CachedRowSet interface, but other implementations
  58  * may choose to implement the JdbcRowSet interface to satisfy
  59  * particular needs.
  60  * 
  61  * After an application has attempted to synchronize a RowSet object with
  62  * the data source (by calling the CachedRowSet
  63  * method acceptChanges), and one or more conflicts have been found,
  64  * a rowset's SyncProvider object creates an instance of
  65  * SyncResolver. This new SyncResolver object has
  66  * the same number of rows and columns as the
  67  * RowSet object that was attempting the synchronization. The
  68  * SyncResolver object contains the values from the data source that caused
  69  * the conflict(s) and null for all other values.
  70  * In addition, it contains information about each conflict.
  71  *
  72  *
  73  * 
Getting and Using a SyncResolver Object

  74  *
  75  * When the method acceptChanges encounters conflicts, the
  76  * SyncProvider object creates a SyncProviderException
  77  * object and sets it with the new SyncResolver object. The method
  78  * acceptChanges will throw this exception, which
  79  * the application can then catch and use to retrieve the
  80  * SyncResolver object it contains. The following code snippet uses the
  81  * SyncProviderException method getSyncResolver to get
  82  * the SyncResolver object resolver.
  83  * 

  84  * {@code
  85  *     } catch (SyncProviderException spe) {
  86  *         SyncResolver resolver = spe.getSyncResolver();
  87  *     ...
  88  *     }
  89  *
  90  * }
  91  * 
  92  * 
  93  * With resolver in hand, an application can use it to get the information
  94  * it contains about the conflict or conflicts.  A SyncResolver object
  95  * such as resolver keeps
  96  * track of the conflicts for each row in which there is a conflict.  It also places a
  97  * lock on the table or tables affected by the rowset's command so that no more
  98  * conflicts can occur while the current conflicts are being resolved.
  99  * 
 100  * The following kinds of information can be obtained from a SyncResolver
 101  * object:
 102  *
 103  *    
What operation was being attempted when a conflict occurred

 104  * The SyncProvider interface defines four constants
 105  * describing states that may occur. Three
 106  * constants describe the type of operation (update, delete, or insert) that a
 107  * RowSet object was attempting to perform when a conflict was discovered,
 108  * and the fourth indicates that there is no conflict.
 109  * These constants are the possible return values when a SyncResolver object
 110  * calls the method getStatus.
 111  * 

 112  *     {@code int operation = resolver.getStatus(); }
 113  * 
 114  *
 115  *    
The value in the data source that caused a conflict

 116  * A conflict exists when a value that a RowSet object has changed
 117  * and is attempting to write to the data source
 118  * has also been changed in the data source since the last synchronization.  An
 119  * application can call the SyncResolver method
 120  * getConflictValue to retrieve the
 121  * value in the data source that is the cause of the conflict because the values in a
 122  * SyncResolver object are the conflict values from the data source.
 123  * 

 124  *     java.lang.Object conflictValue = resolver.getConflictValue(2);
 125  * 
 126  * Note that the column in resolver can be designated by the column number,
 127  * as is done in the preceding line of code, or by the column name.
 128  * 
 129  * With the information retrieved from the methods getStatus and
 130  * getConflictValue, the application may make a determination as to
 131  * which value should be persisted in the data source. The application then calls the
 132  * SyncResolver method setResolvedValue, which sets the value
 133  * to be persisted in the RowSet object and also in the data source.
 134  * 

 135  *     resolver.setResolvedValue("DEPT", 8390426);
 136  * 
 137  * In the preceding line of code,
 138  * the column name designates the column in the RowSet object
 139  * that is to be set with the given value. The column number can also be used to
 140  * designate the column.
 141  * 
 142  * An application calls the method setResolvedValue after it has
 143  * resolved all of the conflicts in the current conflict row and repeats this process
 144  * for each conflict row in the SyncResolver object.
 145  *
 146  *
 147  * 
Navigating a SyncResolver Object

 148  *
 149  * Because a SyncResolver object is a RowSet object, an
 150  * application can use all of the RowSet methods for moving the cursor
 151  * to navigate a SyncResolver object. For example, an application can
 152  * use the RowSet method next to get to each row and then
 153  * call the SyncResolver method getStatus to see if the row
 154  * contains a conflict.  In a row with one or more conflicts, the application can
 155  * iterate through the columns to find any non-null values, which will be the values
 156  * from the data source that are in conflict.
 157  * 
 158  * To make it easier to navigate a SyncResolver object, especially when
 159  * there are large numbers of rows with no conflicts, the SyncResolver
 160  * interface defines the methods nextConflict and
 161  * previousConflict, which move only to rows
 162  * that contain at least one conflict value. Then an application can call the
 163  * SyncResolver method getConflictValue, supplying it
 164  * with the column number, to get the conflict value itself. The code fragment in the
 165  * next section gives an example.
 166  *
 167  * 
Code Example

 168  *
 169  * The following code fragment demonstrates how a disconnected RowSet
 170  * object crs might attempt to synchronize itself with the
 171  * underlying data source and then resolve the conflicts. In the try
 172  * block, crs calls the method acceptChanges, passing it the
 173  * Connection object con.  If there are no conflicts, the
 174  * changes in crs are simply written to the data source.  However, if there
 175  * is a conflict, the method acceptChanges throws a
 176  * SyncProviderException object, and the
 177  * catch block takes effect.  In this example, which
 178  * illustrates one of the many ways a SyncResolver object can be used,
 179  * the SyncResolver method nextConflict is used in a
 180  * while loop. The loop will end when nextConflict returns
 181  * false, which will occur when there are no more conflict rows in the
 182  * SyncResolver object resolver. In This particular code fragment,
 183  * resolver looks for rows that have update conflicts (rows with the status
 184  * SyncResolver.UPDATE_ROW_CONFLICT), and the rest of this code fragment
 185  * executes only for rows where conflicts occurred because crs was attempting an
 186  * update.
 187  * 
 188  * After the cursor for resolver has moved to the next conflict row that
 189  * has an update conflict, the method getRow indicates the number of the
 190  * current row, and
 191  * the cursor for the CachedRowSet object crs is moved to
 192  * the comparable row in crs. By iterating
 193  * through the columns of that row in both resolver and crs, the conflicting
 194  * values can be retrieved and compared to decide which one should be persisted. In this
 195  * code fragment, the value in crs is the one set as the resolved value, which means
 196  * that it will be used to overwrite the conflict value in the data source.
 197  *
 198  * 

 199  * {@code
 200  *     try {
 201  *
 202  *         crs.acceptChanges(con);
 203  *
 204  *     } catch (SyncProviderException spe) {
 205  *
 206  *         SyncResolver resolver = spe.getSyncResolver();
 207  *
 208  *         Object crsValue;  // value in the RowSet object
 209  *         Object resolverValue:  // value in the SyncResolver object
 210  *         Object resolvedValue:  // value to be persisted
 211  *
 212  *         while(resolver.nextConflict())  {
 213  *             if(resolver.getStatus() == SyncResolver.UPDATE_ROW_CONFLICT)  {
 214  *                 int row = resolver.getRow();
 215  *                 crs.absolute(row);
 216  *
 217  *                 int colCount = crs.getMetaData().getColumnCount();
 218  *                 for(int j = 1; j <= colCount; j++) {
 219  *                     if (resolver.getConflictValue(j) != null)  {
 220  *                         crsValue = crs.getObject(j);
 221  *                         resolverValue = resolver.getConflictValue(j);
 222  *                         . . .
 223  *                         // compare crsValue and resolverValue to determine
 224  *                         // which should be the resolved value (the value to persist)
 225  *                         resolvedValue = crsValue;
 226  *
 227  *                         resolver.setResolvedValue(j, resolvedValue);
 228  *                      }
 229  *                  }
 230  *              }
 231  *          }
 232  *      }
 233  * }
 234  * @author  Jonathan Bruce
 235  */

 237 public interface SyncResolver extends RowSet {
 238     /**
 239      * Indicates that a conflict occurred while the RowSet object was
 240      * attempting to update a row in the data source.
 241      * The values in the data source row to be updated differ from the
 242      * RowSet object's original values for that row, which means that
 243      * the row in the data source has been updated or deleted since the last
 244      * synchronization.
 245      */
 246      public static int UPDATE_ROW_CONFLICT = 0;
 247 
 248     /**
 249      * Indicates that a conflict occurred while the RowSet object was
 250      * attempting to delete a row in the data source.
 251      * The values in the data source row to be updated differ from the
 252      * RowSet object's original values for that row, which means that
 253      * the row in the data source has been updated or deleted since the last
 254      * synchronization.
 255      */
 256     public static int DELETE_ROW_CONFLICT = 1;
 257 
 258    /**
 259     * Indicates that a conflict occurred while the RowSet object was
 260     * attempting to insert a row into the data source.  This means that a
 261     * row with the same primary key as the row to be inserted has been inserted
 262     * into the data source since the last synchronization.
 263     */
 264     public static int INSERT_ROW_CONFLICT = 2;
 265 
 266     /**
 267      * Indicates that no conflict occurred while the RowSet object
 268      * was attempting to update, delete or insert a row in the data source. The values in
 269      * the SyncResolver will contain null values only as an indication
 270      * that no information in pertinent to the conflict resolution in this row.
 271      */
 272     public static int NO_ROW_CONFLICT = 3;
 273 
 274     /**
 275      * Retrieves the conflict status of the current row of this SyncResolver,
 276      * which indicates the operation
 277      * the RowSet object was attempting when the conflict occurred.
 278      *
 279      * @return one of the following constants:
 280      *         SyncResolver.UPDATE_ROW_CONFLICT,
 281      *         SyncResolver.DELETE_ROW_CONFLICT,
 282      *         SyncResolver.INSERT_ROW_CONFLICT, or
 283      *         SyncResolver.NO_ROW_CONFLICT
 284      */
 285     public int getStatus();
 286 
 287     /**
 288      * Retrieves the value in the designated column in the current row of this
 289      * SyncResolver object, which is the value in the data source
 290      * that caused a conflict.
 291      *
 292      * @param index an int designating the column in this row of this
 293      *        SyncResolver object from which to retrieve the value
 294      *        causing a conflict
 295      * @return the value of the designated column in the current row of this
 296      *         SyncResolver object
 297      * @throws SQLException if a database access error occurs
 298      */
 299     public Object getConflictValue(int index) throws SQLException;
 300 
 301     /**
 302      * Retrieves the value in the designated column in the current row of this
 303      * SyncResolver object, which is the value in the data source
 304      * that caused a conflict.
 305      *
 306      * @param columnName a String object designating the column in this row of this
 307      *        SyncResolver object from which to retrieve the value
 308      *        causing a conflict
 309      * @return the value of the designated column in the current row of this
 310      *         SyncResolver object
 311      * @throws SQLException if a database access error occurs
 312      */
 313     public Object getConflictValue(String columnName) throws SQLException;
 314 
 315     /**
 316      * Sets obj as the value in column index in the current row of the
 317      * RowSet object that is being synchronized. obj
 318      * is set as the value in the data source internally.
 319      *
 320      * @param index an int giving the number of the column into which to
 321      *        set the value to be persisted
 322      * @param obj an Object that is the value to be set in the
 323      *        RowSet object and persisted in the data source
 324      * @throws SQLException if a database access error occurs
 325      */
 326     public void setResolvedValue(int index, Object obj) throws SQLException;
 327 
 328     /**
 329      * Sets obj as the value in column columnName in the current row of the
 330      * RowSet object that is being synchronized. obj
 331      * is set as the value in the data source internally.
 332      *
 333      * @param columnName a String object giving the name of the column
 334      *        into which to set the value to be persisted
 335      * @param obj an Object that is the value to be set in the
 336      *        RowSet object and persisted in the data source
 337      * @throws SQLException if a database access error occurs
 338      */
 339     public void setResolvedValue(String columnName, Object obj) throws SQLException;
 340 
 341     /**
 342      * Moves the cursor down from its current position to the next row that contains
 343      * a conflict value. A SyncResolver object's
 344      * cursor is initially positioned before the first conflict row; the first call to the
 345      * method nextConflict makes the first conflict row the current row;
 346      * the second call makes the second conflict row the current row, and so on.
 347      * 
 348      * A call to the method nextConflict will implicitly close
 349      * an input stream if one is open and will clear the SyncResolver
 350      * object's warning chain.
 351      *
 352      * @return true if the new current row is valid; false
 353      *         if there are no more rows
 354      * @throws SQLException if a database access error occurs or the result set type
 355      *     is TYPE_FORWARD_ONLY
 356      *
 357      */
 358     public boolean nextConflict() throws SQLException;
 359 
 360     /**
 361      * Moves the cursor up from its current position to the previous conflict
 362      * row in this SyncResolver object.
 363      * 
 364      * A call to the method previousConflict will implicitly close
 365      * an input stream if one is open and will clear the SyncResolver
 366      * object's warning chain.
 367      *
 368      * @return true if the cursor is on a valid row; false
 369      *     if it is off the result set
 370      * @throws SQLException if a database access error occurs or the result set type
 371      *     is TYPE_FORWARD_ONLY
 372      */
 373     public boolean previousConflict() throws SQLException;
 374 
 375 }
